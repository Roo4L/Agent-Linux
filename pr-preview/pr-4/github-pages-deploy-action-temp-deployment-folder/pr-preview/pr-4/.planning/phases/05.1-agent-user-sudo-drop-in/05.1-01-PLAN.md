# Plan 05.1-01: Sudoers drop-in provisioner + bats coverage

## Objective

Deliver the sudoers drop-in provisioner (`20-sudoers.sh`) and bats coverage (`22-agent-sudo.bats`) that realize ADR-012 and close INST-06 + BHV-07 in a single plan.

## Threat Model

- **T-05.1-01 (syntax error in sudoers file breaks system sudo):** Mitigated by `visudo -cf tmpfile` gate before `install` moves the file into place. If the tmpfile doesn't pass `visudo -cf`, the installer exits non-zero without touching `/etc/sudoers.d/`.
- **T-05.1-02 (incorrect permissions on drop-in):** Mitigated by `install -m 0440 -o root -g root` atomic install semantics — mode + ownership set at rename time; never a window where the file exists with wrong permissions.
- **T-05.1-03 (world-readable drop-in exposes policy):** Mode `0440` means only root can read. Agent user cannot `cat /etc/sudoers.d/agentlinux` — can only observe effective policy via `sudo -l`.
- **T-05.1-04 (re-run corrupts the drop-in via double-append):** Mitigated by atomic-overwrite install (not `echo >>`). sha256 byte-stability verified by tests.

## Must-Haves

- `plugin/provisioner/20-sudoers.sh` exists, passes shellcheck + shfmt, starts with `set -euo pipefail`.
- Running the installer produces `/etc/sudoers.d/agentlinux` with mode `0440`, owner `root:root`, containing exactly `agent ALL=(ALL) NOPASSWD: ALL` (plus optional header comment).
- `visudo -cf /etc/sudoers.d/agentlinux` returns exit 0.
- `sudo -u agent sudo -n true` returns exit 0 after install (INST-06).
- Re-running the installer produces a byte-identical `/etc/sudoers.d/agentlinux` (sha256 stable).
- `tests/bats/22-agent-sudo.bats` exists with ≥3 @tests covering INST-06 + BHV-07; every @test cites its req ID.
- `./tests/docker/run.sh ubuntu-22.04` and `./tests/docker/run.sh ubuntu-24.04` both exit 0 — existing bats stay green plus the new sudo suite.
- `bash tests/harness/run.sh` still exits 0.

### Action

Create `plugin/provisioner/20-sudoers.sh` using this exact shape:

```bash
#!/usr/bin/env bash
# 20-sudoers.sh — install /etc/sudoers.d/agentlinux granting passwordless sudo
# to the agent user (scope: ALL commands, per ADR-012).
#
# Runs AFTER 10-agent-user.sh (agent user must exist) and BEFORE 30-nodejs.sh.
# Idempotent: atomic overwrite via install(1), byte-stable on re-run.
# Security: visudo -cf gate before file install — never ship an invalid sudoers.

set -euo pipefail

readonly SUDOERS_FILE="/etc/sudoers.d/agentlinux"
readonly SUDOERS_MODE="0440"
readonly SUDOERS_OWNER="root:root"
readonly SUDOERS_CONTENT="# Installed by AgentLinux — grants passwordless sudo to agent user.
# Scope: ALL commands. See docs/decisions/012-agent-user-full-sudo.md.
agent ALL=(ALL) NOPASSWD: ALL
"

log_info "20-sudoers.sh: installing $SUDOERS_FILE (ADR-012)"

# Ensure parent dir exists with correct ownership + mode.
ensure_dir /etc/sudoers.d 0755 root:root

# Write to tmpfile + visudo -cf gate + atomic install.
tmpfile=$(mktemp)
printf '%s' "$SUDOERS_CONTENT" > "$tmpfile"
if ! visudo -cf "$tmpfile" >/dev/null; then
  log_error "visudo -cf validation failed on tmpfile — refusing to install"
  rm -f "$tmpfile"
  exit 1
fi
install -m "$SUDOERS_MODE" -o root -g root "$tmpfile" "$SUDOERS_FILE"
rm -f "$tmpfile"

# Post-install verify.
if ! visudo -cf "$SUDOERS_FILE" >/dev/null; then
  log_error "post-install visudo -cf failed — sudoers file is corrupt"
  exit 1
fi

log_info "20-sudoers.sh: agent user has passwordless sudo (ALL commands)"
```

Then create `tests/bats/22-agent-sudo.bats`:

```bash
#!/usr/bin/env bats
# 22-agent-sudo.bats — INST-06 + BHV-07 per ADR-012.

load 'helpers/invoke_modes'
load 'helpers/assertions'

# BHV-07: file integrity
@test "BHV-07: /etc/sudoers.d/agentlinux exists with mode 0440 root:root" {
  run test -f /etc/sudoers.d/agentlinux
  assert_exit_zero "BHV-07" "$status" "$output"

  run stat -c '%a %U:%G' /etc/sudoers.d/agentlinux
  assert_exit_zero "BHV-07" "$status" "$output"
  [[ "$output" == "440 root:root" ]] || __fail "BHV-07" "mode=440 owner=root:root" "$output" "N/A"
}

# BHV-07: content exact match
@test "BHV-07: /etc/sudoers.d/agentlinux contains exact NOPASSWD line" {
  run grep -Fx 'agent ALL=(ALL) NOPASSWD: ALL' /etc/sudoers.d/agentlinux
  assert_exit_zero "BHV-07" "$status" "$output"
}

# BHV-07: visudo validation
@test "BHV-07: /etc/sudoers.d/agentlinux passes visudo -cf" {
  run visudo -cf /etc/sudoers.d/agentlinux
  assert_exit_zero "BHV-07" "$status" "$output"
}

# INST-06: agent user can sudo passwordless
@test "INST-06: agent user has passwordless sudo" {
  run_sudo_u 'sudo -n true'
  assert_exit_zero "INST-06" "$status" "$output"
}

# INST-06 + BHV-07: re-run idempotency
@test "BHV-07: sudoers drop-in is byte-stable on re-run" {
  pre=$(sha256sum /etc/sudoers.d/agentlinux | cut -d' ' -f1)
  run bash /agentlinux-src/plugin/bin/agentlinux-install
  [ "$status" -eq 0 ] || __fail "BHV-07" "installer exit 0" "exit $status" "see log"
  post=$(sha256sum /etc/sudoers.d/agentlinux | cut -d' ' -f1)
  [[ "$pre" == "$post" ]] || __fail "BHV-07" "sha256 stable across re-run" "pre=$pre post=$post" "N/A"
}
```

Note: the re-run test assumes the installer source is mounted at `/agentlinux-src/` in the Docker test image (per Phase 2 Plan 02-05 convention); adjust path if the Dockerfile names it differently.
