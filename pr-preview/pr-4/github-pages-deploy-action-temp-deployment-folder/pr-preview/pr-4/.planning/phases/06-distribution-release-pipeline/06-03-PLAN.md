**QEMU command (Pattern 4 lines 473-482):**
```bash
qemu-system-x86_64 \
  -cpu host -enable-kvm \
  -m 2048 -smp 2 \
  -drive "file=${IMG},if=virtio,snapshot=on" \
  -drive "file=${RUN_DIR}/seed.iso,format=raw,readonly=on" \
  -netdev user,id=n0,hostfwd=tcp::2222-:22 \
  -device virtio-net,netdev=n0 \
  -nographic -serial file:"${RUN_DIR}/serial.log" \
  &
QEMU_PID=$!
```
`snapshot=on` on the main drive prevents cache corruption — writes go to a hidden overlay.

**SSH loop with cloud-init wait:**
```bash
SSH_OPTS="-o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null -o ConnectTimeout=5 -i ${RUN_DIR}/id_ed25519 -p 2222"
for i in $(seq 1 60); do
  if ssh $SSH_OPTS root@localhost 'cloud-init status --wait' 2>/dev/null; then
    break
  fi
  sleep 5
done
```
The `cloud-init status --wait` call blocks inside the guest until seed user-data finishes — more reliable than polling nc port 22 alone (port can be open before cloud-init packages installed).

  <name>Task 2: Implement tests/qemu/boot.sh — full cloud-init + QEMU + SSH + installer + bats orchestrator</name>
  <files>tests/qemu/boot.sh</files>
  <action>
    Implement per 06-RESEARCH.md Pattern 4 (lines 414-508) and qemu-harness SKILL.md §"Target boot flow".

    Structure:
    ```bash
    #!/usr/bin/env bash
    # tests/qemu/boot.sh — boot Ubuntu cloud image, ssh-in, run installer + bats.
    # Source: 06-RESEARCH.md Pattern 4 + qemu-harness SKILL.md.
    # Pitfalls mitigated inline: 4 (KVM access), 10 (cloud image sha verify on cache hit),
    # T-06-06 (per-run SSH keypair hygiene).
    set -euo pipefail

    UBUNTU_VERSION=${1:?usage: tests/qemu/boot.sh <22.04|24.04>}

    # 0. Read cloud-images.txt to look up image-url + sha-url for the requested version.
    IMAGES_MANIFEST="$(dirname "$0")/cloud-images.txt"
    LINE=$(grep -E "^${UBUNTU_VERSION//./\\.} " "$IMAGES_MANIFEST" || true)
    [[ -n "$LINE" ]] || { printf 'unsupported Ubuntu version %s (must appear in %s)\n' "$UBUNTU_VERSION" "$IMAGES_MANIFEST" >&2; exit 64; }
    read -r _ IMG_URL SHASUMS_URL <<< "$LINE"

    case "$UBUNTU_VERSION" in
      22.04) RELEASE=jammy ;;
      24.04) RELEASE=noble ;;
      *) echo "unsupported" >&2; exit 64 ;;
    esac

    # 1. Fail-fast on KVM (Pitfall 4 — silent TCG fallback blows past timeout).
    if [[ ! -r /dev/kvm || ! -w /dev/kvm ]]; then
      printf 'ERROR: /dev/kvm not accessible — refusing to run (TCG fallback is 20-30x slower; would exceed timeout)\n' >&2
      printf '       On ubuntu-24.04 GitHub Actions runner, the KVM udev rule step in nightly-qemu.yml must run first.\n' >&2
      exit 1
    fi

    CACHE=${AGENTLINUX_QEMU_CACHE:-$HOME/.cache/agentlinux/qemu}
    mkdir -p "$CACHE"
    IMG_NAME="ubuntu-${UBUNTU_VERSION}-server-cloudimg-amd64.img"
    IMG="$CACHE/$IMG_NAME"
    SHASUMS="$CACHE/${RELEASE}-SHA256SUMS"

    # 2. Download + SHA-verify the cloud image. Always re-verify on cache hit (Pitfall 10).
    if [[ ! -f "$IMG" ]]; then
      printf 'downloading %s\n' "$IMG_URL"
      curl -fsSL -o "$IMG" "$IMG_URL"
    fi
    printf 'refreshing SHA256SUMS manifest\n'
    curl -fsSL -o "$SHASUMS" "$SHASUMS_URL"
    if ! ( cd "$CACHE" && sha256sum --ignore-missing --check "${RELEASE}-SHA256SUMS" ); then
      printf 'ERROR: cloud image SHA256 mismatch — refusing to boot (tampered cache? Re-download by `rm %s`)\n' "$IMG" >&2
      exit 1
    fi

    # 3. Per-run SSH keypair (T-06-06 — keypair generated in mktemp, EXIT trap cleans).
    RUN_DIR=$(mktemp -d -t agentlinux-qemu.XXXXXX)
    chmod 0700 "$RUN_DIR"
    cleanup() {
      if [[ -n "${QEMU_PID:-}" ]]; then kill -TERM "$QEMU_PID" 2>/dev/null || true; wait "$QEMU_PID" 2>/dev/null || true; fi
      rm -rf "$RUN_DIR"
    }
    trap cleanup EXIT INT TERM
    ssh-keygen -t ed25519 -N '' -f "$RUN_DIR/id_ed25519" -C 'agentlinux-qemu' >/dev/null

    # 4. Generate seed ISO from the template files (Task 1).
    local_pubkey=$(cat "$RUN_DIR/id_ed25519.pub")
    sed "s|__AGENTLINUX_QEMU_PUBKEY__|${local_pubkey}|" \
      "$(dirname "$0")/cloud-init/user-data" > "$RUN_DIR/user-data"
    sed "s|__AGENTLINUX_QEMU_INSTANCE_ID__|agentlinux-ci-${RELEASE}-$(date +%s)|" \
      "$(dirname "$0")/cloud-init/meta-data" > "$RUN_DIR/meta-data"
    cloud-localds "$RUN_DIR/seed.iso" "$RUN_DIR/user-data" "$RUN_DIR/meta-data"

    # 5. Boot QEMU backgrounded.
    qemu-system-x86_64 \
      -cpu host -enable-kvm \
      -m 2048 -smp 2 \
      -drive "file=${IMG},if=virtio,snapshot=on" \
      -drive "file=${RUN_DIR}/seed.iso,format=raw,readonly=on" \
      -netdev user,id=n0,hostfwd=tcp::2222-:22 \
      -device virtio-net,netdev=n0 \
      -nographic -serial file:"${RUN_DIR}/serial.log" \
      &
    QEMU_PID=$!

    # 6. Wait for cloud-init to finish — block inside the guest, not just polling nc.
    SSH_OPTS=( -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null -o ConnectTimeout=5 -i "${RUN_DIR}/id_ed25519" -p 2222 )
    printf 'waiting for guest cloud-init status --wait ...\n'
    for i in $(seq 1 60); do
      if ssh "${SSH_OPTS[@]}" root@localhost 'cloud-init status --wait' 2>/dev/null; then
        printf 'cloud-init: done\n'
        break
      fi
      sleep 5
      [[ "$i" == 60 ]] && { printf 'ERROR: cloud-init did not finish in 5min\n' >&2; cat "$RUN_DIR/serial.log" >&2 || true; exit 1; }
    done

    # 7. Build + scp the plugin tarball into the guest.
    # We reuse Plan 06-01's build-release.sh to produce a deterministic tarball.
    # Use a "v0.0.0-qemu" tag so this doesn't interfere with a real release.
    QEMU_TAG="v0.0.0-qemu"
    bash "$(dirname "$0")/../../scripts/build-release.sh" "$QEMU_TAG"
    scp "${SSH_OPTS[@]}" "dist/agentlinux-${QEMU_TAG}.tar.gz" root@localhost:/tmp/

    # Also scp the tests/ + plugin catalog (for bats after install) via a secondary tarball.
    TESTS_TAR="$RUN_DIR/tests.tar.gz"
    tar --create --gzip --file="$TESTS_TAR" tests/
    scp "${SSH_OPTS[@]}" "$TESTS_TAR" root@localhost:/tmp/tests.tar.gz

    # 8. Install AgentLinux in-guest (over SSH).
    ssh "${SSH_OPTS[@]}" root@localhost \
      "mkdir -p /opt/agentlinux-src && cd /opt/agentlinux-src && tar -xzf /tmp/agentlinux-${QEMU_TAG}.tar.gz && tar -xzf /tmp/tests.tar.gz && bash plugin/bin/agentlinux-install"

    # 9. Run bats in-guest — full suite (incl. AGT-02 release-gate 51-*.bats).
    BATS_STATUS=0
    ssh "${SSH_OPTS[@]}" root@localhost \
      'cd /opt/agentlinux-src && bats tests/bats/' || BATS_STATUS=$?

    # 10. Artifacts on fail — scp serial.log + bats output back.
    if [[ "$BATS_STATUS" -ne 0 ]]; then
      mkdir -p tests/qemu/artifacts
      cp "$RUN_DIR/serial.log" "tests/qemu/artifacts/serial-${RELEASE}-$(date +%s).log" || true
      printf 'ERROR: in-guest bats suite exited %d on Ubuntu %s\n' "$BATS_STATUS" "$UBUNTU_VERSION" >&2
    fi

    # 11. Poweroff; trap will reap QEMU_PID.
    ssh "${SSH_OPTS[@]}" root@localhost 'poweroff' 2>/dev/null || true
    wait "$QEMU_PID" 2>/dev/null || true
    QEMU_PID=

    exit "$BATS_STATUS"
    ```

    Then `chmod +x tests/qemu/boot.sh`.

    **Why fail-fast on /dev/kvm (Pitfall 4):** Silent TCG fallback exceeds the 30-min CI timeout. Making the harness fail early-and-loud with a clear message saves hours of "why did my release CI hang for 30 minutes."

    **Why cloud-init status --wait vs. nc -z:** `nc -z localhost 2222` returns success when sshd is listening, but cloud-init's `packages:` step may still be installing bats/jq. Running bats before cloud-init finishes → "bats: command not found" → flaky test. `cloud-init status --wait` blocks inside the guest until ALL seed actions finish.

    **Why snapshot=on:** Every QEMU run writes to an ephemeral overlay, not the cached image. Prevents cache corruption (Pitfall 10's poisoned-cache scenario is defended upstream by SHA verify; snapshot=on is defense-in-depth against accidental mutation).

    **Why scp separate tests/ tarball:** The release tarball contains ONLY `plugin/` (per 06-01 locked decision). The bats tests live at `tests/bats/` — not in the release tarball. QEMU guest needs both to install AND to test.

    **Per-task atomic commit:** `feat(06-03): implement tests/qemu/boot.sh (TST-03 — full cloud-init + QEMU + ssh + installer + bats orchestrator)`.
