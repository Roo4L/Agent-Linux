  <name>Task 1: Create scripts/build-release.sh — version gate + reproducible tarball + SHA256 sidecar + catalog snapshot</name>
  <files>scripts/build-release.sh, .gitignore</files>
  <action>
    Create `scripts/build-release.sh` implementing Pattern 3 from 06-RESEARCH.md (lines 335-405). The script MUST:

    1. **Shebang + safety:** `#!/usr/bin/env bash` + `set -euo pipefail`. No top-level destructive commands before arg parsing.

    2. **Arg parsing + tag validation (per D — locked decision):**
       ```bash
       TAG=${1:?usage: scripts/build-release.sh v<X.Y.Z>[-suffix]}
       if [[ ! "$TAG" =~ ^v[0-9]+\.[0-9]+\.[0-9]+(-[A-Za-z0-9.]+)?$ ]]; then
         printf 'tag %s does not match vX.Y.Z[-suffix]\n' "$TAG" >&2
         exit 64
       fi
       VERSION=${TAG#v}
       ```

    3. **Version-consistency gate (locked — tag must match both package.json + catalog.json):**
       Read `plugin/cli/package.json` `.version` and `plugin/catalog/catalog.json` `.version` via `jq -r`. Fail with clear message if either mismatches `$VERSION`. Three-way lock (tag + CLI pkg + catalog pkg) prevents releasing a tag whose catalog version is stale.

    4. **CLI build (per D — ADR-011 bundle pattern):**
       `(cd plugin/cli && npm install --no-audit --no-fund && npm run build)` — ensures `dist/index.js` is fresh. Tarball must include both `plugin/cli/dist/` (tsc output) and `plugin/cli/node_modules/` (runtime deps) per Phase 4 Plan 04-06 precedent. Do NOT `npm install -g` — per CLAUDE.md critical rule.

    5. **Reproducible tarball (Pitfall 5 mitigation — SOURCE_DATE_EPOCH):**
       ```bash
       mkdir -p dist
       SOURCE_DATE_EPOCH=${SOURCE_DATE_EPOCH:-$(git log -1 --pretty=%ct HEAD)}
       export SOURCE_DATE_EPOCH
       tar \
         --sort=name \
         --owner=0 --group=0 --numeric-owner \
         --mtime="@${SOURCE_DATE_EPOCH}" \
         --pax-option=exthdr.name=%d/PaxHeaders/%f,delete=atime,delete=ctime \
         --create --gzip --file="dist/agentlinux-${TAG}.tar.gz" \
         plugin/
       ```
       The `--sort=name` + numeric-owner=0 + pinned mtime combination is the reproducible-builds.org canonical recipe. Re-running on the same commit MUST produce byte-identical gzip. Note: `gzip` embeds mtime; pass `--mtime=@${SOURCE_DATE_EPOCH}` to `tar`, and verify GNU tar's `--gzip` wraps with a deterministic gzip (no-mtime inner header) — if needed, pipe through `gzip -n` explicitly.

    6. **SHA256 sidecar (GNU default format, Pattern 1):**
       ```bash
       ( cd dist && sha256sum "agentlinux-${TAG}.tar.gz" > "agentlinux-${TAG}.tar.gz.sha256" )
       ```

    7. **Catalog snapshot (CAT-05 — Pitfall 8 mitigation):**
       ```bash
       cp plugin/catalog/catalog.json "dist/catalog-${TAG}.json"
       ```
       Use `cp`, NOT `jq .` — preserves byte-for-byte formatting. sha256 of `dist/catalog-${TAG}.json` MUST equal sha256 of `plugin/catalog/catalog.json` at release time.

    8. **Optional .deb (Pitfall 6 + ADR-006 "optional"):**
       If `SKIP_DEB=1` OR `! command -v fpm`, emit a one-line "skipping .deb (fpm not installed or SKIP_DEB=1)" message and continue. Otherwise:
       ```bash
       fpm -s dir -t deb \
         --name agentlinux --version "$VERSION" \
         --description "Agent-ready Ubuntu environment: correctly-owned Node.js + curated agent catalog" \
         --url https://agentlinux.org --license Apache-2.0 \
         --architecture all --depends curl \
         --after-install packaging/deb/postinst.sh \
         --chdir plugin/ \
         --package "dist/agentlinux_${VERSION}_all.deb" \
         .
       ```

    9. **Final summary line:** `printf 'Built: %s + .sha256 + catalog-%s.json%s\n' "$TARBALL" "$TAG" "${DEB_SUFFIX:-}"` — stdout-only; no emojis.

    10. **Executable bit:** `chmod +x scripts/build-release.sh` after write.

    Also create `scripts/` directory if missing (`mkdir -p scripts`).

    **Update `.gitignore`:** append `dist/` so CI builds never accidentally commit release artifacts. Use `ensure_marker_block` shape if consistent with existing `.gitignore`; otherwise a plain append with comment `# Phase 6: release build output` is fine.

    **Why sort + owner=0 + mtime=@SOURCE_DATE_EPOCH:** Phase 6 research Pitfall 5 — without these, two CI re-runs of the same tag produce different SHA256s. The release-sibling `.sha256` sidecar becomes unverifiable. Reproducibility is also a v0.4+ baseline we should not break now.

    **Why `cp` for catalog snapshot (not `jq .`):** Pitfall 8 — if the `/opt/agentlinux/catalog/<v>/catalog.json` staged by 50-registry-cli.sh drifts from the release-sibling `catalog-v<v>.json`, `agentlinux upgrade` reads divergent data. Byte-for-byte copy at release time is the anti-drift mechanism.

    **Why per-task atomic commit:** CLAUDE.md critical rule. One commit for the script; one for the .gitignore tweak; one for postinst.sh (Task 2); one for the bats test (Task 3).

  <name>Task 2: Create packaging/deb/postinst.sh — minimal .deb postinst hook</name>
  <files>packaging/deb/postinst.sh</files>
  <action>
    Create `packaging/deb/postinst.sh`. Minimal implementation matching ADR-006 "optional .deb":

    ```bash
    #!/usr/bin/env bash
    # Debian postinst hook — invoked after `dpkg -i agentlinux_<X.Y.Z>_all.deb`.
    # Responsibility: invoke the installer entrypoint.
    # NOTE: fpm's --after-install wires this to the package lifecycle; the .deb
    # extracts plugin/ to /opt/agentlinux/ per fpm `--prefix`, then runs this script.
    set -euo pipefail

    INSTALLER=/opt/agentlinux/bin/agentlinux-install
    if [[ ! -x "$INSTALLER" ]]; then
      printf 'postinst: %s missing or not executable; aborting\n' "$INSTALLER" >&2
      exit 1
    fi

    exec "$INSTALLER" "$@"
    ```

    chmod +x packaging/deb/postinst.sh.

    Why exec + not just run: fpm wires this as the package's postinst; exec replaces the dpkg-spawned shell with agentlinux-install so its exit code propagates to dpkg correctly (avoids "dpkg says success but installer actually failed" bugs).

    Why keep this simple: ADR-006 scoping. If fpm proves fragile in CI, we skip .deb entirely via SKIP_DEB=1 (Task 1) and ship only the curl-pipe-bash channel. The postinst is the single point of fpm-specific logic; keeping it minimal reduces surface.

    No tests needed — fpm's own packaging tests + the tarball's bats tests cover the installer behavior; this hook is just the dpkg bridge.
  </action>
