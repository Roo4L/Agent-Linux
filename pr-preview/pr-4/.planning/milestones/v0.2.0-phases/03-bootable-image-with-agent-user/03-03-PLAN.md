  <action>
Add an `environment_vars` attribute to the shell provisioner block in `packer/agentlinux.pkr.hcl` (inside the `build` block, lines 68-75).

The updated provisioner block should be:

```hcl
  provisioner "shell" {
    execute_command  = "echo 'packer' | sudo -S bash -c '{{ .Vars }} {{ .Path }}'"
    environment_vars = ["ONE_CONTEXT_VERSION=${var.one_context_version}"]
    scripts = [
      "scripts/01-base.sh",
      "scripts/02-one-context.sh",
      "scripts/03-cleanup.sh",
    ]
  }
```

This works because:
- `environment_vars` sets env vars that Packer injects via `{{ .Vars }}` in `execute_command`
- `{{ .Vars }}` expands to `ONE_CONTEXT_VERSION=6.10.0-3` (or whatever override) before the script path
- `02-one-context.sh` line 11 reads `ONE_CONTEXT_VERSION="${ONE_CONTEXT_VERSION:-6.10.0-3}"` which will now receive the Packer variable value instead of always falling back to the hardcoded default

Do NOT modify any other file. Do NOT change `02-one-context.sh` -- its fallback default is correct defensive coding. Only `agentlinux.pkr.hcl` needs the `environment_vars` line added.
