  <name>Task 1: Create page foundation with CSS design system and sticky nav</name>
  <files>index.html</files>
  <action>
Create `index.html` with the complete HTML5 boilerplate and CSS design system. This file will be the ONLY HTML file for the entire site (per DSGN-03: static HTML/CSS/JS, no build step).

**HTML head:**
- DOCTYPE, charset UTF-8, viewport meta tag (critical for responsive: `width=device-width, initial-scale=1`)
- Title: "AgentLinux — Linux, for agents"
- Google Fonts: JetBrains Mono (weights 400, 500, 700) with `font-display: swap`
- Preconnect hints for fonts.googleapis.com and fonts.gstatic.com

**Inline `<style>` block — CSS design system:**
- CSS custom properties on `:root`:
  - `--bg-primary: #0a0a0a`, `--bg-secondary: #141414`, `--bg-tertiary: #1e1e1e`
  - `--text-primary: #e0e0e0`, `--text-secondary: #a0a0a0`, `--text-muted: #666666`
  - `--border: #2a2a2a`, `--accent: #ffffff`
- Reset: `* { margin: 0; padding: 0; box-sizing: border-box; }`
- Body: dark background (`--bg-primary`), monospace font stack (`'JetBrains Mono', 'Fira Code', 'Cascadia Code', 'SF Mono', monospace`), `line-height: 1.6`, `font-size: 16px` minimum
- `html`: `scroll-behavior: smooth`, `scroll-padding-top: 80px` (for sticky nav offset)
- `@media (prefers-reduced-motion: reduce)`: disable smooth scroll and animations
- Section base styles: `min-height: 100vh` for major sections, flex column centered, `padding: 6rem 2rem`, `max-width: 900px` centered
- Do NOT apply `min-height: 100vh` to FAQ or footer (they should be content-height per research pitfall #3)

**Sticky nav bar:**
- `<nav>` element with `position: sticky; top: 0; z-index: 1000`
- Background: `--bg-primary` with `border-bottom: 1px solid var(--border)`
- Inner container: flex, space-between, max-width 900px centered
- Left: "AgentLinux" logo text link (font-weight 700, white)
- Right: nav links — Problem, Features, Signup, FAQ — all as anchor links (`#problem`, `#features`, `#signup`, `#faq`)
- Nav link styles: `--text-secondary` color, uppercase, `letter-spacing: 0.05em`, hover to `--accent`
- Mobile responsive: reduce nav link font-size and spacing at 640px breakpoint

**Section placeholders:**
- Add empty `<section>` elements with correct IDs: `#hero`, `#problem`, `#features`, `#comparison`, `#signup`, `#faq`
- Add empty `<footer>` element
- Each placeholder should have a temporary heading showing the section name (will be replaced in later plans)

Per user decision: dark theme with black/white/gray palette ONLY — no accent colors beyond white. Terminal personality through monospace type and code-style elements, but NOT a full terminal simulation. No ASCII art.
