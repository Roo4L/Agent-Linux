**Existing Phase 1 release.yml (to be replaced wholesale):**
- On push tags `v*.*.*` + workflow_dispatch with `tag` input.
- Currently does: Resolve tag → call scripts/build-release.sh with guard → upload via action-gh-release (unpinned v2 — this plan pins to v2.6.2).
- The replacement keeps the same trigger shape; expands the body to the 4-gate pipeline.

**Existing Phase 1 test.yml:**
- Has pre-commit, cli-unit, bats-docker jobs. release.yml's gate-1 can either reuse test.yml (`uses: ./.github/workflows/test.yml`) or duplicate the steps. Pattern 6 in research uses reuse. We'll duplicate gate-2's bats-docker but keep gate-1 as explicit pre-commit + cli-unit (simpler to debug when something fails on a tag).

**Concurrency spec (Pitfall 9 mitigation):**
```yaml
concurrency:
  group: release-${{ github.ref }}
  cancel-in-progress: false  # NEVER cancel an in-progress release
```

**action-gh-release pin (Standard Stack line 115):**
`uses: softprops/action-gh-release@v2.6.2` — last Node-20-compatible release. v3 requires Node-24 runtime.

**files glob (publish job):**
```yaml
files: |
  dist/agentlinux-*.tar.gz
  dist/agentlinux-*.tar.gz.sha256
  dist/catalog-*.json
  dist/agentlinux_*.deb
```
The glob does not fail if `.deb` is missing (fpm-skipped builds produce only 3 files; the glob matches what exists).

**Existing deploy.yml expected structure:**
The repo has an existing v0.1.0 deploy.yml (for agentlinux.org GH Pages). Before editing, grep for it: `grep -l 'github/pages' .github/workflows/*.yml`. If it exists, extend its "build site" step with a `cp packaging/curl-installer/install.sh install.sh` line before the upload-pages-artifact. If deploy.yml does NOT exist (repo is pre-Pages), create it per Pattern 5 (06-RESEARCH.md lines 521-531).

**The `install.sh` at repo root after copy:**
When the developer edits `packaging/curl-installer/install.sh`, deploy.yml re-syncs the root copy on each deploy. This avoids Pitfall 7 (drift between two maintained copies) — the canonical file is under `packaging/`; `install.sh` at root is CI-generated and gitignored OR tracked as a re-synced copy. Choose: **gitignore the root install.sh and let deploy.yml generate it at pages-build time** — simpler than tracking a re-synced copy.

    Replace entire file with the 4-gate orchestration per 06-RESEARCH.md Pattern 6 (lines 544-672). Final shape:

    ```yaml
    name: release
    on:
      push:
        tags: ['v*.*.*']
      workflow_dispatch:
        inputs:
          tag:
            description: 'Release tag (e.g. v0.3.0)'
            required: true

    # Pitfall 9 — never cancel an in-progress release.
    concurrency:
      group: release-${{ github.ref }}
      cancel-in-progress: false

    permissions:
      contents: write

    jobs:
      # Gate 0 — resolve the tag (either from GITHUB_REF or workflow_dispatch input).
      resolve:
        runs-on: ubuntu-24.04
        outputs:
          tag: ${{ steps.tag.outputs.value }}
        steps:
          - uses: actions/checkout@v4
          - id: tag
            env:
              INPUT_TAG: ${{ inputs.tag }}
            run: |
              tag="${INPUT_TAG:-${GITHUB_REF##*/}}"
              if [[ ! "$tag" =~ ^v[0-9]+\.[0-9]+\.[0-9]+(-[A-Za-z0-9.]+)?$ ]]; then
                echo "::error::invalid tag '$tag' — must match vX.Y.Z[-suffix]"
                exit 1
              fi
              echo "value=$tag" >> "$GITHUB_OUTPUT"

      # Gate 1 — pre-commit + Node unit tests. Fastest gate; fails cheapest.
      gate-1-precommit:
        needs: resolve
        runs-on: ubuntu-24.04
        steps:
          - uses: actions/checkout@v4
          - uses: actions/setup-python@v5
            with: { python-version: '3.12' }
          - uses: actions/setup-node@v4
            with: { node-version: '22' }
          - name: Install pre-commit
            run: pip install pre-commit==4.0.1
          - name: Run pre-commit on all files
            run: pre-commit run --all-files --show-diff-on-failure
          - name: CLI unit tests
            working-directory: plugin/cli
            run: |
              npm install --no-audit --no-fund
              npm test

      # Gate 2 — Docker matrix. bats in tests/bats/ incl. 51-*.bats (TST-05 inside Docker).
      gate-2-docker:
        needs: gate-1-precommit
        runs-on: ubuntu-24.04
        timeout-minutes: 20
        strategy:
          fail-fast: false
          matrix:
            ubuntu: [ubuntu-22.04, ubuntu-24.04]
        steps:
          - uses: actions/checkout@v4
          - name: Run Docker bats matrix (incl. AGT-02 release-gate 51-*.bats — TST-05 inside Docker)
            run: bash tests/docker/run.sh ${{ matrix.ubuntu }}

      # Gate 3 — QEMU matrix. cloud-image boot + installer + bats (incl. 51-*.bats — TST-05 inside QEMU).
      gate-3-qemu:
        needs: gate-2-docker
        runs-on: ubuntu-24.04
        timeout-minutes: 45
        strategy:
          fail-fast: false
          matrix:
            ubuntu: ['22.04', '24.04']
        steps:
          - uses: actions/checkout@v4
          - uses: actions/setup-node@v4
            with: { node-version: '22' }
          # Pitfall 4 — KVM udev rule on hosted ubuntu-24.04 runner.
          - name: Enable /dev/kvm access (Pitfall 4)
            run: |
              sudo bash -c 'echo "KERNEL==\"kvm\", GROUP=\"kvm\", MODE=\"0666\"" > /etc/udev/rules.d/99-kvm.rules'
              sudo udevadm control --reload-rules
              sudo udevadm trigger --name-match=kvm
              [[ -r /dev/kvm && -w /dev/kvm ]] || { echo "::error::/dev/kvm not accessible"; exit 1; }
          - name: Install QEMU + cloud-image-utils
            run: |
              sudo apt-get update
              sudo apt-get install -y --no-install-recommends qemu-system-x86 cloud-image-utils netcat-openbsd
          - name: Cache Ubuntu cloud images
            uses: actions/cache@v4
            with:
              path: ~/.cache/agentlinux/qemu
              key: cloud-image-${{ matrix.ubuntu }}-${{ hashFiles('tests/qemu/cloud-images.txt') }}
          - name: QEMU boot + installer + bats (incl. 51-*.bats — TST-05 inside QEMU)
            run: bash tests/qemu/boot.sh ${{ matrix.ubuntu }}
          - name: Upload QEMU artifacts on failure
            if: failure()
            uses: actions/upload-artifact@v4
            with:
              name: qemu-artifacts-${{ matrix.ubuntu }}
              path: tests/qemu/artifacts/
              if-no-files-found: ignore
              retention-days: 14

      # Gate 4 — pinned-combo (TST-08). Install every catalog agent at its pinned_version
      # and run 50-agents.bats + 51-*.bats (AGT-02) in Ubuntu 24.04 Docker (our pinned dev env).
      # Docker 24.04 bats already does this end-to-end (per Phase 5 Plan 05-04); the job
      # re-runs it in release context as a separate observable gate in the Actions UI.
      gate-4-pinned-combo:
        needs: gate-3-qemu
        runs-on: ubuntu-24.04
        timeout-minutes: 30
        steps:
          - uses: actions/checkout@v4
          - name: Install pinned catalog combo + run bats (TST-08)
            # tests/docker/run.sh ubuntu-24.04 installs all pinned agents via catalog.json
            # and runs tests/bats/ incl. 50-agents.bats + 51-*.bats — satisfies TST-08.
            run: bash tests/docker/run.sh ubuntu-24.04

      # Build — only after all 4 gates green. Reproducible tarball + sha256 + catalog snapshot + optional .deb.
      build:
        needs: [resolve, gate-4-pinned-combo]
        runs-on: ubuntu-24.04
        timeout-minutes: 15
        steps:
          - uses: actions/checkout@v4
          - uses: actions/setup-node@v4
            with: { node-version: '22' }
          - name: Install fpm (optional .deb — Pitfall 6)
            run: |
              sudo apt-get update
              sudo apt-get install -y --no-install-recommends ruby-dev build-essential
              sudo gem install --no-document fpm || { echo "::warning::fpm install failed — proceeding with SKIP_DEB=1"; echo "SKIP_DEB=1" >> "$GITHUB_ENV"; }
          - name: Build release artifacts
            run: bash scripts/build-release.sh ${{ needs.resolve.outputs.tag }}
          - name: Verify .sha256 sibling exists (T-06-08 mitigation)
            run: |
              tag="${{ needs.resolve.outputs.tag }}"
              test -s "dist/agentlinux-${tag}.tar.gz" \
                && test -s "dist/agentlinux-${tag}.tar.gz.sha256" \
                && test -s "dist/catalog-${tag}.json" \
                || { echo "::error::release artifact assembly failed — missing required files in dist/"; ls -la dist/; exit 1; }
              ( cd dist && sha256sum -c "agentlinux-${tag}.tar.gz.sha256" )
          - uses: actions/upload-artifact@v4
            with:
              name: release-artifacts
              path: dist/
              if-no-files-found: error

      # Publish — only on tag push (not dispatch). softprops/action-gh-release@v2.6.2 (Node-20 pinned).
      publish:
        needs: [resolve, build]
        if: startsWith(github.ref, 'refs/tags/v')
        runs-on: ubuntu-24.04
        permissions:
          contents: write
        steps:
          - uses: actions/download-artifact@v4
            with:
              name: release-artifacts
              path: dist/
          - uses: softprops/action-gh-release@v2.6.2
            with:
              tag_name: ${{ needs.resolve.outputs.tag }}
              generate_release_notes: true
              files: |
                dist/agentlinux-*.tar.gz
                dist/agentlinux-*.tar.gz.sha256
                dist/catalog-*.json
                dist/agentlinux_*.deb
    ```

    Key rationale notes (put as inline comments in the yaml too):
    - `concurrency { cancel-in-progress: false }` — Pitfall 9 mitigation.
    - `fail-fast: false` in every matrix — both Ubuntu versions' failures should be visible.
    - Gate 4 using `tests/docker/run.sh ubuntu-24.04` (not a separate script) — the Docker image already installs pinned combo via Phase 4+5 provisioners; this is a semantic gate, not a re-implementation.
    - `softprops/action-gh-release@v2.6.2` — pin is explicit per Standard Stack; protects against Node-24 requirement drift.
    - `workflow_dispatch` dry-runs DO NOT publish (the `if: startsWith(github.ref, 'refs/tags/v')` guard on the publish job). Build still produces artifacts; you can download them from the Actions UI.
    - `actions/download-artifact@v4` mirrors upload-artifact@v4 (match major; Actions v2-v3 compat broke).

    Per-task atomic commit: `feat(06-04): rewrite release.yml — 4-gate release pipeline + pinned publish (TST-05, TST-08, INST-03, CAT-05)`.
