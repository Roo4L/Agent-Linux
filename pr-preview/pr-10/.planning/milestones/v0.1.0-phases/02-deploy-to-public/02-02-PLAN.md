<task type="auto">
  <name>Task 1: Create GitHub Actions workflow and CNAME file</name>
  <files>.github/workflows/deploy.yml, CNAME</files>
  <action>
1. Create `.github/workflows/deploy.yml` with the following content:

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: ["main"]
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: "pages"
  cancel-in-progress: false

jobs:
  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
      - name: Setup Pages
        uses: actions/configure-pages@v5
      - name: Upload artifact
        uses: actions/upload-pages-artifact@v4
        with:
          path: '.'
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

Key points:
- `path: '.'` uploads the entire repo root (index.html is at root, no build step per DSGN-03)
- `workflow_dispatch` allows manual re-deploys from GitHub UI
- `concurrency` prevents overlapping deployments
- Single job since there is no build step
- Trigger is push to main only (no PR previews per user decision)

2. Create `CNAME` file at repo root with a single line:
```
agentlinux.org
```
No trailing newline, no protocol prefix, no path -- just the bare domain.

  <how-to-verify>
Complete these steps in order:

**Step 1: Enable GitHub Pages (GitHub UI or CLI)**

Option A -- GitHub UI:
1. Go to https://github.com/Roo4L/Agent-Linux/settings/pages
2. Under "Build and deployment" > Source, select "GitHub Actions"
3. Save

Option B -- gh CLI:
```bash
gh api -X PUT "/repos/Roo4L/Agent-Linux/pages" -f build_type=workflow
```

**Step 2: Push to trigger first deployment**
Push the current branch (with the workflow and CNAME) to main. The GitHub Actions workflow will run and deploy the site.

**Step 3: Set custom domain in GitHub Pages settings**
1. Go to https://github.com/Roo4L/Agent-Linux/settings/pages
2. Under "Custom domain", enter `agentlinux.org` and click Save
3. GitHub will verify DNS (may show "DNS check in progress")

Or via CLI:
```bash
gh api -X PUT "/repos/Roo4L/Agent-Linux/pages" -f cname=agentlinux.org
```

**Step 4: Configure DNS on Hostinger**
Add these A records pointing to GitHub Pages:

| Type | Host | Value |
|------|------|-------|
| A | @ | 185.199.108.153 |
| A | @ | 185.199.109.153 |
| A | @ | 185.199.110.153 |
| A | @ | 185.199.111.153 |

Optional IPv6 AAAA records:

| Type | Host | Value |
|------|------|-------|
| AAAA | @ | 2606:50c0:8000::153 |
| AAAA | @ | 2606:50c0:8001::153 |
| AAAA | @ | 2606:50c0:8002::153 |
| AAAA | @ | 2606:50c0:8003::153 |

**Step 5: Wait for DNS propagation (15 min to 48 hours)**
Check progress:
```bash
dig agentlinux.org +short
```
Should return GitHub Pages IPs (185.199.108-111.153).

**Step 6: Enable HTTPS**
1. Go to https://github.com/Roo4L/Agent-Linux/settings/pages
2. Check "Enforce HTTPS" (may take up to 24 hours for certificate provisioning)

**Step 7: Verify**
```bash
curl -sI https://agentlinux.org | head -5
```
Should return HTTP/2 200.
