  <name>Task 1: Build email signup form, FAQ section, and footer</name>

  <action>
Replace the `#signup`, `#faq`, and `<footer>` placeholder sections in `index.html`.

**Email Signup Section (`#signup`):**
- Located before FAQ (per user decision: "single dedicated section near bottom of page, before FAQ")
- Section heading: e.g., "Get early access" or "Be the first to know" (Claude drafts)
- Brief subtext: 1 sentence about what they're signing up for (early access to AgentLinux)
- Form structure (per research Pattern 5 — Buttondown progressive enhancement):
  ```html
  <form action="https://buttondown.com/api/emails/embed-subscribe/agentlinux" method="post" class="embeddable-buttondown-form" id="signup-form">
    <input type="email" name="email" placeholder="you@example.com" required />
    <input type="hidden" name="embed" value="1" />
    <button type="submit">Join the waitlist</button>
  </form>
  ```
- Use `agentlinux` as placeholder Buttondown username (per research open question #1 — user will confirm actual username)
- Form styling: email input and button on same line (flexbox), input has dark background (`--bg-tertiary`), white border, monospace font. Button styled like the hero CTA (white border, transparent bg, white text, hover fills white with black text).
- Success/error state styling: `.success` class for post-submit message, `.error` class for error message

**Progressive enhancement JS** (add to the inline `<script>` block):
- Intercept form submit with `addEventListener`
- Check `window.location.protocol === 'file:'` — if so, let form POST normally (redirect)
- Otherwise: `e.preventDefault()`, disable button, show "Submitting..." text
- `fetch()` to form action with form data as URL-encoded body
- On success (response.ok or response.redirected): replace form innerHTML with success message: "You're on the list. We'll be in touch."
- On error: restore button, fall back to `form.submit()` (normal POST with redirect)
- This handles the CORS uncertainty gracefully (per research open question #3)

**FAQ Section (`#faq`):**
- Located after signup
- Section heading: "Frequently asked questions" or similar
- 3-5 questions with brief answers (per user decision: "short and punchy, 1-2 sentences each")
- Suggested questions (Claude drafts final versions):
  1. What is AgentLinux? — A Linux distribution purpose-built for running AI coding agents.
  2. When will it be available? — We're in early development. Join the waitlist to be notified.
  3. Is it free? — Yes, AgentLinux will be free and open source.
  4. What agents does it support? — Any agent that runs on Linux — Claude Code, Codex, Aider, and more.
  5. How is this different from Docker? — AgentLinux is a full OS, not a container. Agents get a persistent, dedicated environment.
- Style each Q&A as a simple div with bold question and paragraph answer (NOT `<details>/<summary>` — keep it simple and always visible)
- Do NOT apply `min-height: 100vh` to FAQ — let height be content-driven (per research pitfall #3)
- Use `--bg-primary` background

**Footer:**
- Simple footer element, NOT full-viewport height
- Content: copyright line ("(c) 2026 AgentLinux" or similar), optionally a link to GitHub
- `--bg-secondary` background, `--text-muted` text, smaller font size
- Padding: `2rem` top/bottom
- Centered text
