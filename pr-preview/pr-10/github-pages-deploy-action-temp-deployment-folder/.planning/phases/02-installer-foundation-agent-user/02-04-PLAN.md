1. `/etc/profile.d/agentlinux.sh` → BHV-06 (interactive login) + BHV-05 (`sudo -u agent -i`)

Phase 3 will extend `/etc/profile.d/agentlinux.sh` with the npm-global prefix; Phase 2 pre-installs `/home/agent/.local/bin` so any agent-owned scripts placed there are already on PATH across all six modes.

Output: One provisioner file. Verification in this plan is shellcheck + file-content grep; end-to-end six-mode matrix verification (interactive bash login, non-interactive SSH, cron, systemd, sudo -u, sudo -u -i) lands in Plan 05's Docker harness + bats suite.

    Target shape (~80-100 lines):

    ```bash
    #!/usr/bin/env bash
    # plugin/provisioner/40-path-wiring.sh — six-mode PATH/locale wiring.
    # Sourced by plugin/bin/agentlinux-install. Inherits set -euo pipefail + traps + tee.
    # Requirements: BHV-02 (non-interactive SSH), BHV-03 (cron), BHV-04 (systemd),
    #               BHV-05 (sudo -u agent / sudo -u agent -i), BHV-06 (interactive login).
    #
    # Four artefacts. Each invocation mode reads env from a different source.
    # RESEARCH Pattern 4 supplies the exact content; here we commit it behind
    # `install -m` for installer-owned files and `ensure_marker_block --top` for
    # /home/agent/.bashrc (user-owned; we must preserve user content outside marker).
    #
    # Path ordering: agent-owned prefixes FIRST (precedence), system prefixes last.
    # Only paths under /home/agent are agent-owned; /usr/local/bin is root-owned
    # standard. This prevents T-02-08 path-injection by a wider-writable directory.
    # Phase 3 will prepend $HOME/.npm-global/bin here.

    log_info "40-path-wiring: starting"

    # Ensure /home/agent/.local/bin exists so Phase 2 PATH prefix is not a
    # dangling reference before the user ever writes anything there.
    ensure_dir /home/agent/.local 0755 agent:agent
    ensure_dir /home/agent/.local/bin 0755 agent:agent

    # ---------------------------------------------------------------
    # Artefact 1: /etc/profile.d/agentlinux.sh
    # BHV-06 (interactive login) + BHV-05 (sudo -u agent -i login variant).
    # Sourced by /etc/profile → /etc/profile.d/*.sh on login shells.
    # Guard prevents double-append if the file is re-sourced in the same session.
    # ---------------------------------------------------------------
    install -m 0644 /dev/stdin /etc/profile.d/agentlinux.sh <<'PROFILE'
    # AgentLinux login environment (generated by agentlinux-install).
    # Sourced by /etc/profile on interactive login shells AND `sudo -u agent -i`.
    # Re-source guard: bails on second source in the same shell session.
    [ -n "${AGENTLINUX_PROFILE_SOURCED:-}" ] && return
    export AGENTLINUX_PROFILE_SOURCED=1

    export LANG="${LANG:-C.UTF-8}"
    export LC_ALL="${LC_ALL:-C.UTF-8}"

    # Prepend /home/agent/.local/bin to PATH if not already there.
    # Phase 3 extends this with $HOME/.npm-global/bin.
    case ":${PATH}:" in
      *:/home/agent/.local/bin:*) : ;;
      *) PATH="/home/agent/.local/bin:${PATH}" ;;
    esac
    export PATH
    PROFILE
    log_info "wrote /etc/profile.d/agentlinux.sh"

    # ---------------------------------------------------------------
    # Artefact 2: /home/agent/.bashrc marker block at TOP.
    # BHV-02 (non-interactive SSH: `ssh agent@host cmd`) + BHV-05 non-login (`sudo -u agent bash -c`).
    # MUST be at TOP so it runs before the skel /.bashrc's
    #   case $- in *i*) ;; *) return;; esac
    # early-return (Pitfall 2). Otherwise non-interactive bash skips the block.
    # ---------------------------------------------------------------
    # If /home/agent/.bashrc doesn't exist yet (no skel copied), create it empty
    # so ensure_marker_block has a file to operate on. `install -m 0644` creates.
    if [[ ! -f /home/agent/.bashrc ]]; then
      install -m 0644 -o agent -g agent /dev/null /home/agent/.bashrc
    fi
    ensure_marker_block /home/agent/.bashrc "agentlinux-path" --top <<'BASHRC'
    # AgentLinux PATH guard — sources /etc/profile.d/agentlinux.sh unconditionally
    # so non-interactive bash invocations (ssh host 'cmd', sudo -u agent bash -c)
    # see the correct PATH + locale BEFORE the skel /.bashrc case-return guard.
    if [ -f /etc/profile.d/agentlinux.sh ]; then
      . /etc/profile.d/agentlinux.sh
    fi
    BASHRC
    chown agent:agent /home/agent/.bashrc
    chmod 0644 /home/agent/.bashrc
    log_info "wrote agentlinux-path marker block to /home/agent/.bashrc (--top)"

    # ---------------------------------------------------------------
    # Artefact 3: /etc/agentlinux.env
    # BHV-04 (systemd User=agent via EnvironmentFile=/etc/agentlinux.env).
    # FORMAT: KEY=VALUE, NO shell expansion, NO `export`. systemd and cron both
    # parse this shape literally. Pitfall 4: never use `$PATH` substitutions.
    # ---------------------------------------------------------------
    install -m 0644 /dev/stdin /etc/agentlinux.env <<'ENVFILE'
    PATH=/home/agent/.local/bin:/usr/local/bin:/usr/bin:/bin
    LANG=C.UTF-8
    LC_ALL=C.UTF-8
    ENVFILE
    log_info "wrote /etc/agentlinux.env (systemd EnvironmentFile + cron header template)"

    # ---------------------------------------------------------------
    # Artefact 4: /etc/cron.d/agentlinux
    # BHV-03 (cron). The PATH= line at the top of a cron.d file applies to every
    # job below it. Phase 2 ships NO default jobs — this is the PATH contract
    # future jobs inherit. Pitfall 4: literal PATH only, no `$PATH` expansion.
    # ---------------------------------------------------------------
    install -m 0644 /dev/stdin /etc/cron.d/agentlinux <<'CRON'
    # AgentLinux cron environment (generated by agentlinux-install).
    # Any agent cron job placed in this file inherits PATH/locale below.
    # Phase 2 ships NO default jobs. Example shape (do NOT uncomment):
    #   0 3 * * * agent /usr/bin/true

    PATH=/home/agent/.local/bin:/usr/local/bin:/usr/bin:/bin
    LANG=C.UTF-8
    LC_ALL=C.UTF-8
    CRON
    log_info "wrote /etc/cron.d/agentlinux (PATH + locale header; no default jobs)"

    log_info "40-path-wiring: done (four artefacts written)"
    ```

    Invariants:
    - NO raw `echo "..." >> /etc/...` anywhere. Installer-owned files use `install -m 0644 /dev/stdin <<EOF`. `/home/agent/.bashrc` uses `ensure_marker_block` because the user may have other content there.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| root installer → /etc/profile.d/agentlinux.sh | Root-owned, world-readable; sourced by every login shell on the host. Content must not execute untrusted logic. |
| root installer → /etc/agentlinux.env | Root-owned; parsed literally (no shell expansion) by systemd EnvironmentFile and cron. |
| root installer → /etc/cron.d/agentlinux | Root-owned; cron parses PATH= header literally. |
| root installer → /home/agent/.bashrc | agent-owned; partial edit via marker-block; user content outside block survives re-runs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02 | Tampering | /home/agent/.bashrc re-run | mitigate | `ensure_marker_block --top` replaces ONLY content between `agentlinux-path` markers; re-runs are byte-idempotent. No blind `echo >>` anywhere in the provisioner. |
| T-02-08 | Elevation of Privilege | PATH ordering | mitigate | PATH order is `/home/agent/.local/bin:/usr/local/bin:/usr/bin:/bin` — agent-owned prefix FIRST but ONLY under `/home/agent` (agent user's own dir). A wider-writable directory earlier on PATH would enable shadow attacks; this ordering places the agent-owned prefix first with the agent as sole writer, and excludes any world-writable or group-writable paths. |
| T-02-09 | Tampering | /etc/profile.d/agentlinux.sh re-run | mitigate | `install -m 0644 /dev/stdin` is an atomic full-file overwrite. Content is fixed source (no env-variable substitution by the provisioner), so re-runs produce byte-identical files. Re-source guard `AGENTLINUX_PROFILE_SOURCED` inside the file prevents shell-session double-append during runtime. |
| T-02-10 | Tampering | /etc/cron.d/agentlinux re-run | mitigate | `install -m 0644 /dev/stdin` atomic overwrite; content is fixed; byte-idempotent on re-run. PATH literal expanded at author-time (not runtime) — Pitfall 4 mitigation. |
| T-02-11 | Information Disclosure | /etc/agentlinux.env | accept | File is mode 0644 (world-readable); contains no secrets in Phase 2 (just PATH + LANG + LC_ALL). Phase 3+ MUST NOT add secrets to this file — Phase 3 plan-review will enforce. |
| T-02-12 | Elevation of Privilege | sudoers drop-in absent | mitigate | CONTEXT locks: zero sudoers drop-in in Phase 2. Provisioner writes nothing to `/etc/sudoers.d/*`. Verification: `grep -q 'sudoers.d' plugin/provisioner/40-path-wiring.sh` returns empty. |
</threat_model>
