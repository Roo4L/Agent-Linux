    ```bash
    # Appended in Phase 3 (Plan 03-02).
    #
    # RT-04 gate. Input is the bats `$output` populated by a prior `run` or
    # `invoke_mode` that executed `npm config get prefix`. Passes if the prefix
    # starts with `/home/agent/` (trailing slash prevents a hypothetical
    # `/home/agent-staging/...` match). Fails every other shape with a TST-04
    # four-line diagnostic naming expected-vs-observed prefix and the likely
    # source (`~agent/.npmrc`) the debugger should read.
    #
    # T-03-07 mitigation: catches a regression where npm resolves the prefix to
    # /usr, /usr/local, or any non-agent-owned path across any INVOKE_MODE.
    #
    # Usage (note the req-id carries the mode suffix for multi-mode loops):
    #   invoke_mode "$mode" 'npm config get prefix'
    #   assert_user_prefix_in_home "RT-04 (${mode})"
    # ---------------------------------------------------------------------------
    assert_user_prefix_in_home() {
      local req_id=$1
      local observed
      observed=$(printf '%s' "${output:-}" | tr -d '[:space:]')

      case "$observed" in
        /home/agent/*)
          return 0
          ;;
        *)
          __fail "$req_id" \
            "npm config get prefix under /home/agent/" \
            "observed: ${observed:-<empty>}" \
            "~agent/.npmrc (expected: prefix=/home/agent/.npm-global)"
          ;;
      esac
    }
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| test harness → agent user | Bats tests run as root inside the Docker/QEMU container and invoke `sudo -u agent -H bash --login -c` to run npm as agent. Any test that forgets the `-u agent` prefix and runs npm as root re-introduces the Phase 2 Pitfall-2 root-owned global modules bug. |
| test harness → npm registry | `npm install -g cowsay@1.6.0` fetches from registry.npmjs.org over HTTPS. Integrity via npm's sha-512 metadata (embedded in package.json). Accepted upstream; pinning `@1.6.0` prevents version drift. |
| cowsay package → filesystem | cowsay's own install creates two bins (`cowsay` + `cowthink`) + a lib directory. A future cowsay version shipping a `postinstall` script with filesystem side-effects could leave artefacts outside `/home/agent/.npm-global/` — RT-03's byte-clean assertion is our only detection. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05 | Tampering / PATH shadow | Test assertion that cowsay binary resolves to agent-owned path | mitigate | `assert_path_has "RT-02 (${mode})" "/home/agent/.npm-global/bin/cowsay"` pins resolution to the agent-owned path. Any `/usr/local/bin/cowsay` shim would fail this assertion because the Phase 3 PATH order (npm-global FIRST — see Plan 03-01 Task 2 Pitfall 4 mitigation) resolves to the agent-owned binary first. Even IF a shim existed, the assertion would catch a resolution regression. |
| T-03-06 | Tampering / Uninstall residue | RT-03 byte-clean filesystem assertion | mitigate | Assert BOTH `/home/agent/.npm-global/bin/cowsay` AND `/home/agent/.npm-global/bin/cowthink` are absent (Pitfall 9 — cowsay@1.6.0 ships TWO bin entries). Also assert `/home/agent/.npm-global/lib/node_modules/cowsay` directory is absent. Three filesystem checks + six-mode `command -v cowsay` PATH absence checks = strong mutation-kill. |
| T-03-07 | Information disclosure / Config confusion | `npm config get prefix` returning `/usr` in any mode | mitigate | `assert_user_prefix_in_home` (new helper — Task 1) fires a TST-04 diagnostic if prefix starts with anything other than `/home/agent/`.
