<task type="auto">
  <name>Task 1: Build the QCOW2 image with Packer</name>
  <files>output/agentlinux-0.2.0-amd64.qcow2</files>
  <action>
Run the Packer build to produce the QCOW2 image.

Steps:
1. Ensure QEMU plugin is initialized: `/usr/bin/packer init packer/`
2. Run the build: `/usr/bin/packer build packer/`
   - This downloads the Debian 12 genericcloud QCOW2 (~329 MB)
   - Boots the VM with KVM acceleration
   - Cloud-init creates the packer user from cd_content
   - Packer SSHs in and runs the three provisioner scripts
   - VM shuts down, Packer compresses the output
   - Expected duration: ~2-3 minutes with KVM acceleration
3. Verify output exists: `test -f output/agentlinux-0.2.0-amd64.qcow2`
4. Check image info: `qemu-img info output/agentlinux-0.2.0-amd64.qcow2` -- should show format qcow2

If the build fails:
- SSH timeout: Check that cd_content cloud-init config is correct. The Debian genericcloud image expects NoCloud datasource with cidata label.
- Disk space error during provisioning: Verify disk_size = "10G" in template and that 01-base.sh runs growpart + resize2fs.
- KVM error: Verify /dev/kvm is accessible. Try `accelerator = "tcg"` as fallback (much slower).

IMPORTANT: Use `/usr/bin/packer` to avoid cracklib-packer at /usr/sbin/packer.
  </action>

<task type="auto">
  <name>Task 2: Verify image boots and one-context is installed</name>
  <files>(no files modified -- verification only)</files>
  <action>
Verify the built image is correctly configured by inspecting it with QEMU and qemu-nbd/guestfish.

**Boot verification:**
Run a headless QEMU instance with a timeout to confirm the image boots:
```
timeout 60 qemu-system-x86_64 \
  -enable-kvm \
  -m 1024 \
  -cpu host \
  -drive file=output/agentlinux-0.2.0-amd64.qcow2,format=qcow2,if=virtio \
  -nographic \
  -serial mon:stdio \
  -no-reboot 2>&1 | tee /tmp/boot-test.log
```
The output should show kernel boot messages and reach a login prompt (or at least show systemd reaching multi-user.target). Check `/tmp/boot-test.log` for:
- Kernel version line (confirms boot started)
- "login:" prompt or systemd "Reached target" messages (confirms boot completed)
- No kernel panics or critical errors

Note: The VM will not have networking in this test (no OpenNebula context CD), so one-context will log warnings about missing context -- that is expected. The key check is that the OS boots.

**Image content verification using virt-customize or qemu-nbd:**
If `virt-customize` (from libguestfs-tools) is available:
```
sudo yum install -y libguestfs-tools-c 2>/dev/null || true
virt-customize -a output/agentlinux-0.2.0-amd64.qcow2 \
  --run-command 'dpkg -l one-context' \
  --run-command '! dpkg -l cloud-init 2>/dev/null' \
  --run-command 'systemctl is-enabled one-context' \
  --run-command '! getent passwd packer'
```

If virt-customize is not available, use qemu-nbd:
```
sudo modprobe nbd max_part=8
sudo qemu-nbd --connect=/dev/nbd0 output/agentlinux-0.2.0-amd64.qcow2
sudo mkdir -p /mnt/qcow2
sudo mount /dev/nbd0p1 /mnt/qcow2
# Check one-context is installed
sudo chroot /mnt/qcow2 dpkg -l one-context
# Check cloud-init is NOT installed
sudo chroot /mnt/qcow2 dpkg -l cloud-init 2>&1 | grep -q "no packages found" || echo "WARNING: cloud-init still present"
# Check packer user does NOT exist
sudo chroot /mnt/qcow2 getent passwd packer && echo "FAIL: packer user still exists" || echo "OK: packer user removed"
# Check one-context services are enabled
sudo chroot /mnt/qcow2 systemctl is-enabled one-context
sudo chroot /mnt/qcow2 systemctl is-enabled one-context-local
# Cleanup
sudo umount /mnt/qcow2
sudo qemu-nbd --disconnect /dev/nbd0
```

All checks must pass:
- one-context package IS installed
- cloud-init package is NOT installed
- packer user does NOT exist
- one-context and one-context-local services are enabled
