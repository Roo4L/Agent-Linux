```bash
# nodejs.sh rhel arm — key on `nodesource` substring (live-verify exact string on almalinux:9, Phase 19)
# Repo-file presence is probed via nodesource_repo_paths (pkg.sh) so the detect gate, the
# idempotency gate (30-nodejs), and the purge cleanup (agentlinux-install) stay in lockstep.
ns_version=$(rpm -q --qf '%{VERSION}-%{RELEASE}\n' nodejs 2>/dev/null || true)
ns_repo_present=0
while IFS= read -r f; do [[ -f "$f" ]] && ns_repo_present=1; done < <(nodesource_repo_paths)
if [[ "$ns_version" == *nodesource* ]] && [[ "$ns_repo_present" -eq 1 ]]; then
  entries+=("$(__det_nodejs_entry nodesource /usr/bin/node "$ns_version" "$user" /usr)")
fi
# user.sh probe branch — keep field name:
case "${AGENTLINUX_DISTRO_FAMILY:-debian}" in
  rhel) probe=/usr/bin/dnf;     probe_arg=--version ;;
  *)    probe=/usr/bin/apt-get; probe_arg=--help ;;
esac
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| pre-existing host state → detection probes | A brownfield host's rpm DB / repo files / node binary drive classification, then install decisions |
| detect → sudo -u user -n probe | The probe runs a binary as the target user with NOPASSWD; a PATH-shadowed binary could escalate |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-16 | Elevation | sudo-capability probe binary (dnf on rhel) | mitigate | Anchor to the ABSOLUTE path `/usr/bin/dnf` (not bare `dnf`): a user with NOPASSWD for the absolute path plus an earlier-PATH writable dir cannot shadow it — preserves the user.sh:41-47 security rationale for the new probe |
| T-18-17 | Tampering | detection writing state (dnf cache) breaking read-only invariant | mitigate | Use `rpm -q` + file tests only; never a bare `dnf` (which writes /var/cache/dnf). CI grep guard `grep -c 'dnf ' detect/nodejs.sh` is 0 |
| T-18-18 | Spoofing | mis-keying NodeSource classification → wrong reuse decision | mitigate | Dual gate (substring `nodesource` in rpm release AND the yum repo file present) before classifying as NodeSource; AppStream-module is a distinct class — the exact release string is live-verified on almalinux:9 (Phase 19) before locking |
| T-18-19 | Tampering | renaming can_sudo_apt breaking the DET-01 contract | mitigate | Field/export/accessor names frozen; only the probe binary branches; acceptance grep asserts the absence of `can_sudo_pkg` |
