<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| /etc/os-release → detect_distro | Host-supplied ID/VERSION_ID strings drive a branch and are interpolated into log messages |
| remote user → curl-installer pre-gate | The pre-gate is the first code a `curl … \| bash` user hits; a permissive gate would run the staged installer on an unsupported/unsafe host |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-01 | Tampering | distro_detect.sh ID matching | mitigate | Match `ID` exactly (case arm), never `ID_LIKE`; Rocky/RHEL/CentOS/Fedora/Alma 8/10 explicitly rejected — no silent admission of an untested family |
| T-18-02 | Injection | ID/VERSION_ID interpolated into log_error | mitigate | Values only ever reach `log_*` (printf-style) and `[[ ]]`/`case` comparisons — never `eval` or an unquoted command word; keep `"${ID:-}"` quoting |
| T-18-03 | Elevation | curl-installer lockstep gate drift | mitigate | Keep the two gates byte-identical in allowlist; the 60-curl-installer.bats fixture exercises this path so divergence is caught before release |
| T-18-04 | Tampering | AGENTLINUX_OS_RELEASE_PATH seam | accept | Test-only override; defaults to /etc/os-release; a caller able to set it already has the privilege to write /etc/os-release — no new exposure |
</threat_model>
