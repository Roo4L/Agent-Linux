<objective>
Wave 3 — INST-02 idempotency snapshot generalization + REUSE-01 family-token seed. Two
small, well-bounded EL-08/PAR-01 edits: (1) replace the hardcoded
`/etc/apt/sources.list.d/nodesource.sources` literal in `10-installer.bats`'s INST-02
`find` snapshot list (TWO sites: lines 78 and 106) with the family-correct NodeSource
repo path so `find` does not exit 1 on the missing Debian path on EL9; (2) seed
`AGENTLINUX_DISTRO_FAMILY` in `13-reuse.bats`'s lib-chain so the REUSE-01 live
`detect::run_once` probe selects the `dnf` arm on EL9 instead of defaulting to
`apt-get` and reporting `can_sudo_apt=false`.

Purpose: both are root-caused in RESEARCH. INST-02 fails because `find` exits 1 when the
Debian repo path is absent on EL9; REUSE-01 fails because the test sources
`distro_detect.sh` but never calls `detect_distro`, leaving the family token unset so the
product probe (`plugin/lib/detect/user.sh:53`, `case "${AGENTLINUX_DISTRO_FAMILY:-debian}"`)
defaults to the missing `/usr/bin/apt-get`. These are the genuine "assert the same
observable on EL9, never weaken" generalizations. Where a product lib is already sourced
(the 13-reuse lib-chain), reuse the product verb `nodesource_repo_paths`
(`plugin/lib/pkg.sh:145`, already family-correct) rather than re-hardcoding; use the
container-side `distro_nodesource_repo_paths` only in 10-installer where no product lib is
loaded.
Output: 10-installer INST-02 snapshots green on EL9 (needs `diffutils` from Wave 1 for the
`diff`); 13-reuse REUSE-01 live-probe tests green at parity with Ubuntu.
