# Quick Plan 260509-kn2 — AI deslop reviewer + one-time deslop pass

Implements [AL-35](https://copiedwonder.atlassian.net/browse/AL-35). Two
sibling deliverables:

1. A project-scoped `ai-deslop` review subagent that the `/review` loop spawns
   for every changed source file in `plugin/`, `tests/`, `packaging/`, and
   `docs/` (excluding bats specs, vendored files, ADRs, and `.planning/`).
2. A one-time deslop pass that removes existing AI slop from recently
   merged code on `master` — primarily the dogfood harness (`tests/docker/
   dogfood.sh`, `tests/docker/Dockerfile.dogfood`) and the version-lockstep
   wave (`plugin/cli/src/version.ts`, `scripts/check-version-lockstep.sh`).

## Tasks

### Task 1 — Create the `ai-deslop` review subagent

- File: `.claude/agents/ai-deslop.md`
- Action: write the agent following the `bash-engineer.md` / `security-engineer.md`
  layout: frontmatter (`name`, `description`, `tools: Read, Grep, Glob, Bash`),
  `When to spawn`, `What to look for`, `Common gotchas`, free-form output
  format. Rubric mirrors AL-35 description.
- Verify: file exists, frontmatter parses, no `BLOCK/FLAG/PASS` rigid output.
- Done: `head -5 .claude/agents/ai-deslop.md` shows the frontmatter and the
  description starts with "Reviews changed code for AI-generated slop".

### Task 2 — Wire `ai-deslop` into the review loop

- Files:
  - `.claude/skills/review/SKILL.md` (Dispatch table + reviewer list)
  - `CLAUDE.md` (§Review Loop file-type → reviewers table)
- Action: add `ai-deslop` to every source-file row except bats specs, ADRs,
  and `.planning/`. Update the SKILL.md "Related" footer to list the new
  agent file. Update the SKILL.md description to include `ai-deslop` in the
  agents-list.
- Verify: `grep -n ai-deslop .claude/skills/review/SKILL.md CLAUDE.md` shows
  hits in both files.
- Done: dispatch table reads naturally — `ai-deslop` listed alongside the
  domain reviewer for each file pattern.

### Task 3 — One-time deslop pass on recently merged code

- Scope (from `git log --oneline origin/master | head -10`):
  - `tests/docker/dogfood.sh` (227 lines added in b84a4f2 — high slop risk)
  - `tests/docker/Dockerfile.dogfood`
  - `plugin/cli/src/version.ts` (47 lines added in 28fd5a9)
  - `scripts/check-version-lockstep.sh` (85 lines added in 149f2a6)
  - `plugin/provisioner/20-sudoers.sh` (recent additions)
- Action: read each file, apply the `ai-deslop` rubric, remove unambiguous
  slop (tour-guide comments, banner blocks, defensive try/catch around
  trusted internal calls, `any`-cast workarounds, redundant validation,
  filler doc prose). Behaviour-preserving only — no functional change.
- Verify:
  - `pre-commit run --all-files` passes after the deslop commits.
  - `pnpm --dir plugin/cli test` (and/or `pnpm --dir plugin/cli check`) green.
  - `bash -n` clean for every touched shell script; `shellcheck --shell=bash`
    clean (or the pre-existing baseline).
- Done: each touched file is shorter and reads more like the rest of the
  codebase. No ADR or behaviour-test assertion changes.

## Out of scope

- Running QEMU release-gate harness (Docker matrix locally is sufficient
  for the deslop pass — this is comment-only / dead-code removal).
- Behaviour-test assertion edits (the spec is the spec).
- Auto-removing the new agent from CLAUDE.md if a review nit asks for a
  smaller surface — leave that for the review loop to settle.

## must_haves

- New `ai-deslop` agent file (Task 1).
- Dispatch wiring in SKILL.md and CLAUDE.md (Task 2).
- At least one deslop commit removing real slop on the `worktree-ai-deslop`
  branch, with `pre-commit run --all-files` green (Task 3).
