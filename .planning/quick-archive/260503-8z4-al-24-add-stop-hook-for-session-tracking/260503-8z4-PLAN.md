Topology decision — (a) two sibling Stop entries:
  - Each reminder gets its own entry with matcher "*" and one command.
  - Why (a) over (b) "two commands in the same hooks[] array":
      * Independently togglable (comment one out without disturbing the other)
      * Matches the "separate concerns" idiom used elsewhere in the project
      * Anthropic's docs say both are valid; (a) is more diff-friendly for
        future hooks (no merge conflict on the inner array)
  - Both commands fire on every Stop event; each has its own one-shot guard.
    Worst case = two extra reminder turns when both blocks fire on the same
    stop, both within their per-hook stop_hook_active budget.

Claude Code Stop-hook input shape (stdin, JSON):

  "session_id": "...",
  "transcript_path": "...",
  "stop_hook_active": true | false
}

Stop-hook output (stdout, JSON) when blocking:
{ "decision": "block", "reason": "<text shown to Claude>" }

Exit 0 with no stdout = pass through (let Claude stop).

**File 1: `.claude/hooks/session-tracker-reminder.sh`** (new, mode 0755)

Mirror the shape of `.claude/hooks/review-reminder.sh` exactly. Only the
reason text changes.

```bash
#!/usr/bin/env bash
# Stop hook: reminds Claude to update Jira ticket status via the
# session-tracker skill before stopping.
#
# Behavior:
#   - Reads the Stop-hook JSON envelope from stdin.
#   - If "stop_hook_active": true is present, exits 0 (one-shot guard so the
#     reminder fires at most once per turn — see ADR-010 Refinement
#     2026-05-02; this hook is the second instance of that permitted pattern).
#   - Otherwise emits {"decision":"block","reason":"..."} on stdout. Claude
#     reads the reason, decides whether the session has a Jira ticket worth
#     transitioning, performs the transition (or not), then re-requests stop,
#     at which point stop_hook_active=true and we pass through.
#
# Smoke tests:
#   echo '{"stop_hook_active":false}' | bash .claude/hooks/session-tracker-reminder.sh
#     -> stdout contains '"decision":"block"', exit 0
#   echo '{"stop_hook_active":true}'  | bash .claude/hooks/session-tracker-reminder.sh
#     -> no stdout, exit 0
set -euo pipefail

input=$(cat)

if printf '%s' "$input" | grep -q '"stop_hook_active"[[:space:]]*:[[:space:]]*true'; then
  exit 0
fi

cat <<'JSON'
{"decision":"block","reason":"Before stopping: did this session change the state of a Jira ticket? Use the session-tracker skill to keep the board honest. Trigger conditions: opened a PR/MR -> move ticket to In Review; reviewer pushed back -> move to Reopened (or In Progress); blocked on something external -> move to Blocked and leave a comment; scoped a new deliverable that doesn't fit the current ticket -> file a Sub-task under the parent; PR merged -> mark Done. Skip and request stop again if any of: no Jira ticket is linked to this session, this was a research-only/planning-only/no-code session, or the ticket status was already updated earlier this turn."}
JSON
```

After writing, run `chmod 0755 .claude/hooks/session-tracker-reminder.sh`.
