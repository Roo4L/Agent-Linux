<objective>
Disable Claude Code's background auto-updater for the AgentLinux `agent` user so the catalog's `pinned_version` is actually honored at runtime (AL-51).

Purpose: Today the native installer enables a background updater that drifts `~agent/.local/bin/claude` away from the pinned version the moment the agent first launches `claude`. ADR-011's stability-first invariant relies on `pinned_version` holding both at install time and afterward. The in-recipe AGT-02b assertion (install.sh:47-50) only catches drift at install time, not after — so we need an explicit suppression mechanism. Anthropic's documented `DISABLE_AUTOUPDATER=1` env flag in `~/.claude/settings.json` is the official switch and disables only the background path; manual `claude update` (AGT-02) still works.

Output: install.sh stamps the settings flag (idempotent, merge-safe); uninstall.sh strips it symmetrically (CLI-04); a new bats @test pins the behavior; `docs/internals/claude-code.md` gets a short note in `## What AgentLinux does` (dev-docs-auditor convention).
</objective>

**install.sh** — after the existing post-install AGT-02b version-lock assertion (after line 51 `echo "claude-code: install complete (AGT-02b version-lock satisfied)"`), append a settings-stamping block that disables Claude Code's background auto-updater per AL-51. The block must:

1. Define `settings_file="${AGENTLINUX_AGENT_HOME}/.claude/settings.json"` and `settings_dir="${AGENTLINUX_AGENT_HOME}/.claude"`. Ensure the directory exists with `mkdir -p "${settings_dir}"` (the agent home is pre-created by provisioner; `~/.claude/` may not yet exist on a fresh install).
2. If `settings_file` does not exist: write a fresh `{"env":{"DISABLE_AUTOUPDATER":"1"}}` via `jq -n` to a temp file in `${settings_dir}` then `mv` it into place atomically. Example:
   ```bash
   tmp="${settings_file}.tmp.$$"
   jq -n '{env:{DISABLE_AUTOUPDATER:"1"}}' > "${tmp}"
   mv "${tmp}" "${settings_file}"
   ```
3. If `settings_file` already exists: deep-merge `{"env":{"DISABLE_AUTOUPDATER":"1"}}` into it without clobbering other top-level keys or other keys inside `env`. Use `jq '. + {env: ((.env // {}) + {DISABLE_AUTOUPDATER:"1"})}'` reading from the existing file, writing to the temp file, then `mv`. This preserves theme/model/mcpServers/etc. at the top level and any user-set env vars inside `env`.
4. The `mkdir -p` + temp-file + `mv` flow is the atomic-write pattern from the `<interfaces>` block. Never write directly to `settings_file`.
5. Emit a final `echo "claude-code: settings.json stamped (DISABLE_AUTOUPDATER=1 — AL-51)"`.
6. Do NOT add explicit `chown` — the recipe runs as the agent user via `as_user` dispatch (see install.sh:5 header comment), so ownership is correct by construction.
7. Add one short WHY comment above the block (one paragraph max) explaining: this disables only the background updater; manual `claude update` still works (AGT-02); ADR-011's pinned_version stays honored at runtime. Reference AL-51. Do not narrate the code (no comment narration per CLAUDE.md).
8. Leave install.sh lines 1-53 untouched. The AGT-02b version-lock assertion at lines 47-50 must continue to fail-fast on install-time drift.

**uninstall.sh** — between the existing rm calls (after line 18 `rm -rf "${AGENTLINUX_AGENT_HOME}/.claude/downloads"` and before the "Intentionally NOT removed" comment block at line 20), insert a symmetric settings-strip block:

2. If `settings_file` does not exist, no-op (matches the `rm -f` idempotency idiom used elsewhere in this file).
3. If it exists:
   - Compute the stripped JSON: `jq 'if (.env | type) == "object" then .env = (.env | del(.DISABLE_AUTOUPDATER)) else . end | if (.env == {}) then del(.env) else . end' "${settings_file}"` — drops `env.DISABLE_AUTOUPDATER` and drops `env` itself if it becomes empty after the strip.
   - Capture the stripped output. If the result is an empty object `{}` (test with `jq -e 'length == 0'`), delete the whole `settings_file` (we created it; nothing of the user's remains). Otherwise atomically replace via temp-file + `mv`.
4. The `if` chain must tolerate malformed JSON gracefully — wrap the jq call in an `if jq ... > "${tmp}" 2>/dev/null; then ... else rm -f "${tmp}"; fi` so a corrupted user file does not break uninstall (idempotent + non-fatal).
5. Add one short WHY comment above the block explaining symmetric removal per CLI-04 / AL-51. Match the existing comment voice in this file (terse, WHY-not-WHAT).
6. Leave the "Intentionally NOT removed" warning block at lines 20-22 untouched — `~/.claude/` and `~/.claude.json` are still user data we preserve.

The block must not depend on a sentinel file or marker — the "did we create settings.json?" question is answered structurally by "is the file empty after stripping our key?".
