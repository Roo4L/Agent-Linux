# Quick task 260526-84p: Interactive bats helpers + AGT-02d (AL-54) Summary

**AL-54:** Two pieces of test infrastructure land together — an `expect`-based bats helper for driving interactive Claude Code sessions, and the QEMU host-to-guest forwarding of `ANTHROPIC_API_KEY` that completes the test-secrets pipeline. One behavioral test consumes them: AGT-02d, a red-green pair that asserts AL-51's `DISABLE_AUTOUPDATER` stamp suppresses the in-tool auto-updater across a 90s idle interactive session (PHASE 1 control without the stamp observes drift; PHASE 2 with the stamp observes none).

## What changed

1. **`tests/bats/helpers/interactive.bash`** (NEW, 75 lines) — thin bash wrapper exposing three functions to bats files via `load 'helpers/interactive'`:
   - `claude_login` — drives `claude /login` once per file via `claude-login.exp`; key crosses via `sudo -E` (subprocess env), never argv.
   - `claude_idle_for <seconds>` — holds an interactive `claude` session for N seconds via `claude-idle.exp`.
   - `claude_interactive_run` — loud-error stub (no consumer today; deferred per "avoid ceremony").
   Mirrors `helpers/secrets.bash` + `helpers/assertions.bash` invariants (no `set -euo pipefail` at top, no `eval`, no command-substitution on env var names).

2. **`tests/bats/helpers/expect/claude-login.exp`** (NEW, 75 lines) — standalone expect script. `log_user 0` at entry. Reads `ANTHROPIC_API_KEY` from env (not argv). Tolerant `-re` prompt match (`[Aa]pi[ _-]?[Kk]ey|[Pp]aste.*key|sk-`) with a 30s per-prompt timeout that exits 1 loud on upstream prompt rewording. Sends key + `\r`, expects a success signal, sends Ctrl-D.

3. **`tests/bats/helpers/expect/claude-idle.exp`** (NEW, 41 lines) — standalone expect script. `log_user 0` at entry. Argv 1 = idle seconds (default 90). Spawns `claude`, sleeps INSIDE expect (keeps the pty attached), sends Ctrl-D, expects EOF.

4. **`tests/bats/51-cc-no-autoupdate.bats`** (NEW, 161 lines) — AGT-02d behavioral test (single `@test`, red-green pair). `setup_file` gates on `require_secret ANTHROPIC_API_KEY` first, then recovers the installer state (re-runs `agentlinux-install` when the symlink is absent), re-installs claude-code at the catalog pin via `--force`, and runs `claude_login` once. `teardown_file` re-installs `--force` to restore the stamp.

5. **`tests/bats/50-agents.bats`** (MOD) — AGT-02c docstring at line 184 picks up a four-line cross-ref naming AGT-02d as its behavioral pair. The `@test` body is unchanged.

6. **`tests/docker/Dockerfile.ubuntu-{22,24,26}.04`** (MOD) — `expect` added to the apt-get install list (between `dbus` and `jq`). Header comment notes the package is required by the interactive bats helpers under `tests/bats/helpers/expect/`.

7. **`tests/qemu/cloud-init/user-data`** (MOD) — `expect` added to the `packages:` list. New `write_files:` block dropping `/etc/ssh/sshd_config.d/10-agentlinux-test-acceptenv.conf` with `AcceptEnv ANTHROPIC_API_KEY` (scoped to the single allowlisted variable, not `AcceptEnv *`). `runcmd:` extended with `systemctl restart ssh` so the drop-in is picked up before the first SSH dispatch from `boot.sh`.

8. **`tests/qemu/boot.sh`** (MOD) — `-o SendEnv=ANTHROPIC_API_KEY` added inline at the bats-run `ssh` invocation (around line 401). `SSH_OPTS` array left unchanged. Surrounding comment block names the boundary (workflow `env:` → ssh `SendEnv` → guest sshd `AcceptEnv` → bats process) and pointers at `docs/internals/test-secrets.md`.

9. **`docs/internals/test-interactive.md`** (NEW, 132 lines) — four-section dev-docs entry per the `docs/internals/` contract (`## The problem`, `## What AgentLinux does`, `## Value vs the naive approach`, `## Related`) plus an optional `## Worked example` covering the add-a-new-.exp checklist and pty-debugging steps. ## Deviations from Plan

### 1. [Procedural] `bash -n` removed from the .bats verify gate

- **Found during:** Task 2 verification.
- **Issue:** The plan's verify block called `bash -n tests/bats/51-cc-no-autoupdate.bats`. Bats files use the bats DSL (`@test "..." { ... }`) which does not parse under plain bash — the closing `}` is a syntax error. Confirmed by running `bash -n` against the existing `tests/bats/50-agents.bats` and `tests/bats/51-agt02-release-gate.bats`, both of which also fail with the same error.
- **Resolution:** Skipped the `bash -n` gate for the .bats file (kept it for the .bash helper and boot.sh, where it's meaningful). The structural-grep gates (load-statements, helper-name greps, RED/GREEN phase markers, anti-leak greps) cover the discovery surface. Live discovery via `bats --count` is the right gate; it runs post-commit inside the Docker / QEMU harnesses.
- **Files modified:** None — purely a verification-gate adjustment.
- **Commit:** N/A (no source change).

### 2. [Rule 2 - Missing critical functionality] AGT-02d catalog-pinned-version gate added

- **Found during:** Task 2 authoring.
- **Issue:** The plan's `@test` body computed `pinned=$(jq -r ... .pinned_version ...)` but never asserted on it. Without an assertion the variable is dead code, and a corrupt or missing catalog row would silently flow into the test body.
- **Resolution:** Added a trailing `[[ -n "$pinned" ]] || __fail "AGT-02d (catalog gate)" ...` assertion to materialize the variable and fail loud if the catalog row is missing. Tiny, low-risk, matches the AGT-02b in-recipe assertion shape.
- **Files modified:** `tests/bats/51-cc-no-autoupdate.bats`.
- **Commit:** Included in Task 2 commit `13c5811`.

No other deviations. The plan's structural design (standalone `.exp` scripts + thin bash wrapper, red-green PHASE 1 / PHASE 2 pair, AcceptEnv drop-in via cloud-init `write_files:`, SendEnv inline at the bats-run hop only) was followed verbatim.
