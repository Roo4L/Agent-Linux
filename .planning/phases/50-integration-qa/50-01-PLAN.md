<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| QA skill instructions → agent execution | The reusable workflow can cause installs, removals, and external tool invocations in a disposable environment. |
| Catalog recipes → co-installed agent homes | Recipes write PATH/config/MCP state across multiple agent-owned directories and must not clobber sibling/user state. |
| Credentials → smoke process | Provider secrets may be present at runtime but must never be written to the skill, report, catalog, image, or command transcript. |
| Docker evidence → maintainer handback | Docker cannot prove systemd-user daemon behavior; the report must preserve that boundary. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-50-01 | Tampering | QA scope/report | mitigate | Require direct/adjacent buckets, exact repro/evidence, quiet-round stop record, and explicit untested coverage. |
| T-50-02 | Information disclosure | skill/report/smoke env | mitigate | Keep credentials environment-only; static self-check rejects credential assignment patterns; report records names/status, never values. |
| T-50-03 | Tampering | cross-agent config/PATH | mitigate | Test both install orders, sibling-preserving remove, agent-owned paths, and residue after teardown in fresh sandboxes. |
| T-50-04 | Repudiation | Docker/QEMU handback | mitigate | Record harness, terminal geometry, invocation modes, QEMU-only paths, and explicit maintainer hand-off. |
</threat_model>
