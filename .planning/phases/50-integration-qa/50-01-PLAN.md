    Define per-package evidence as install, command/path/version/ownership, category-specific real operation beyond help/version, remove, post-remove residue, sibling preservation, and forbidden /usr/local shim checks. Give concrete operation examples: authenticated tiny prompts for coding agents; local page interaction for Playwright; seeded usage parsing for ccusage; temp-project scaffolding for spec-kit; hook and representative command flow for rtk; fixture scans for Trivy/Gitleaks; read-only authenticated VCS/Sentry operations; MCP registration plus client-visible keyless or authenticated read-only tool use.

    Define a credential inventory checkpoint. Ask the user for provider/model, GitHub, GitLab, Sentry, Slack, Linear, Atlassian, and any other credentials required by selected operations; inject at runtime only; redact all evidence. An unexpected credential request blocks that idea and triggers a request to the user; help/version or a keyless registration check is not a substitute.

    Define observation-only behavior: reproduce, document, classify, and preserve evidence; do not fix source, recipe, test, or documentation findings during the campaign. Define finding fields: ID, severity, direct/adjacent scope, affected package/workflow, exact repro, redacted evidence, first-seen idea, known/new/unconfirmed-observation classification, disposition, and residual risk.

    Define productive-time accounting: active execution, observation, analysis, and reproduction count; chat idle, usage-limit pauses, waiting for user input, and external blocks do not. A long-running idea counts as clean only after clean completion. A newly reproducible issue resets both measures. A reproduction of an already-known issue does not reset either measure and does not count as clean: the counter measures discovery of new issues, not mere completion. Expected negative behavior counts clean; blocked/incomplete ideas count neither. Stop only when both thresholds hold since the latest new issue, and allow free-form invocation text to override the thresholds without introducing variables.

    End the skill with a Markdown handback template containing the scenario ledger, activity intervals, findings table, known-issue links, blocked ideas, exclusions, credentials/status by class, Docker distro coverage, PTY evidence, cleanup results, and a Coverage limits section.
