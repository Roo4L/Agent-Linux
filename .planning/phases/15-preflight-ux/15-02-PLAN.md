**Why alt-user gate runs BEFORE collect_all_decisions:** the install-user identity is upstream of every other decision (npm-prefix path is `~<user>/.npm-global`, sudoers target is `<user> ALL=(ALL)`, agent installs run via `sudo -u <user>`). Re-running detect::run_once with the new user re-populates every DETECT_* export so the resolution map is correctly oriented from the start. The alternative (asking later and retro-fitting) requires touching every detect/reuse function — undesirable scope.

**reuse::user_decision is extended NOT to add a new return token but to set DETECT_USER_BAIL_REASON.** The existing four-token return surface ({reuse, create, remediate, bail}) is the Phase 13 contract; widening it would force Plan 15-02 to revisit 14-01's gate_or_bail + every provisioner case-branch. Instead, the bail token's REASON is communicated via an export, consumed by Plan 15-02's main() branch.

**Non-TTY behavior (D-15-08):** the same incompatible-user condition exits 65 with a literal hint message that names the conflicting attribute (`wrong-shell`, `home-unwritable`, `name-mismatch`) and suggests `--user=<SUGGESTED>` as the resolution. The SUGGESTED value is computed via the same numeric-suffix scan used by the TTY prompt — consistent advice across modes.

**TOCTOU on useradd (T-15-02-06):** between picking `agent2` and the eventual `useradd -m agent2` call in Plan 14-02's 10-agent-user.sh, another process could create `agent2`. Mitigation: 10-agent-user.sh's useradd is the atomicity boundary (the kernel rejects duplicate uids/names); on collision, the installer re-scans /etc/passwd for the next free name + re-prompts ONCE; if that re-pick also collides, bail with exit 65. Single retry caps the retry loop.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| /etc/passwd → numeric-suffix scan | A malicious /etc/passwd entry could spoof "agent99" to force AgentLinux into exhaustion-bail (T-15-02-04) |
| Operator-typed alt user name → validate_user_name regex | Operator could type a shell-injection sequence; regex must reject (T-15-02-05) |
| Alt-user accepted → useradd <name> | TOCTOU race between scan and useradd (T-15-02-06) |
| reuse::user_decision return + DETECT_USER_BAIL_REASON export → main() gate | Mismatch (returns "bail" without setting reason; or sets a stale reason) could mis-route — defensive checks required |
| Non-TTY bail-with-hint output → operator copy-pastes the suggested name | The suggested name is computed via the SAME numeric-suffix scan that TTY uses, so the operator gets consistent advice |

       ```bash
       # remediate::find_alt_user_name
       #
       # Scan /etc/passwd for the lowest N >= 2 such that agent<N> does NOT
       # already exist. Prints "agent<N>" on stdout. Caps at N=99 — if all
       # are taken, prints empty string and returns 1 (caller bails with a
       # suggest-different-name message per T-15-02-04).
       #
       # Pure-read function — never mutates /etc/passwd. Safe to call during
       # detection.
       remediate::find_alt_user_name() {
         local n
         for ((n = 2; n <= 99; n++)); do
           if ! getent passwd "agent${n}" >/dev/null 2>&1; then
             printf 'agent%d' "$n"
             return 0
           fi
         done
         # Exhaustion (T-15-02-04): all of agent2..agent99 taken.
         printf ''
         return 1
       }

       # remediate::validate_user_name <name>
       #
       # Validates an operator-supplied user name against POSIX-friendly rules:
       #   - Starts with a lowercase letter
       #   - Contains only lowercase letters, digits, underscore, or hyphen
       # Returns 0 on valid, 1 on invalid.
       #
       # T-15-02-05 mitigation: rejects ALL shell metachars (;, |, &, $, etc.).
       # The accepted name is later passed to useradd argv-literally — no shell
       # interpolation possible regardless, but this is the documented contract
       # consumed by every downstream component.
       remediate::validate_user_name() {
         local name=${1:-}
         [[ -n "$name" ]] || return 1
         [[ "$name" =~ ^[a-z][a-z0-9_-]*$ ]] || return 1
         return 0
       }
       ```

       ```bash
           fi

         fi

             printf 'Suggested alternate name: %s\n' "$suggested"
           else
             printf 'No auto-suggested name available (agent2..agent99 all taken).\n'
           fi

         local tries=0
         while [[ $tries -lt 3 ]]; do
           if [[ -n "$suggested" ]]; then
             printf 'Press Enter to use "%s", or type another name: ' "$suggested" >&2
           else
             printf 'Type a name for the new install user: ' >&2
           fi

             printf '\n' >&2
             log_error "[ALT-USER] declined — exiting 65 (EOF on prompt)"
             exit 65
           fi
           # Accept Enter as "use suggested" iff suggested is set.

             chosen="$suggested"
             break
           fi
           if remediate::validate_user_name "$response"; then
             chosen="$response"
             break
           fi
           tries=$((tries + 1))
           printf 'invalid name: %q — must match ^[a-z][a-z0-9_-]*$\n' "$response" >&2
         done

         if [[ -z "$chosen" ]]; then
           log_error "[ALT-USER] 3 invalid responses — exiting 64 EX_USAGE"
           exit 64
         fi

         log_info "[ALT-USER] accepted: ${chosen}"
         export INSTALL_USER="$chosen"
         return 0
       }
       ```
