LOCKED jq construction discipline (T-12-02 mitigation):
- `--arg name value` for ALL string values (jq quotes the value as a JSON string regardless of bytes — newlines, quotes, unicode escapes; see RESEARCH §Don't Hand-Roll table row 1).
- `--argjson name value` for booleans, numbers, integers, and pre-built JSON arrays.
- For booleans built from bash strings: `--argjson present "$([[ \"$DETECT_USER_PRESENT\" == \"true\" ]] && echo true || echo false)"`. The `$(...)` MUST emit literally `true` or `false` — never the bash-string value. (RESEARCH §Pattern 1 § lines 280-300 + verified live in dev env.)
- For arrays of strings (e.g. user.groups): build via `jq -n --arg s "$DETECT_USER_GROUPS" '($s | split(" "))'` OR pre-construct the JSON array with `printf '%s\n' $groups | jq -R . | jq -s .` and pass with --argjson.
- For optional / absent fields: pass the literal JSON `null` via `--argjson` when the bash var is empty: `--argjson mode "$([[ -n \"$DETECT_SUDOERS_MODE\" ]] && jq -nR --arg v \"$DETECT_SUDOERS_MODE\" '$v' || echo null)"`. Simpler shape — pre-build a per-component fragment via the existing per-detector probe (Plans 12-01/02 already wrote each fragment as an object); detect::render_json can `jq -s 'add'` on the cache file plus a wrapper that nests it under `components`.
- NEVER `eval`, NEVER `printf '"%s"' "$var"` for JSON construction.
- NEVER `bash -c "echo {…}"` with interpolation.

LOCKED key-sort idiom (T-12-02 byte-stability):
- Final emit goes through `jq -S '.'` (capital -S). RESEARCH §Open Question 4 verified
  in dev env: `jq -S '.'` sorts keys recursively and writes deterministic byte
  output for a fixed input. Critical for Phase 13's REUSE-02 test that may
  re-run detection multiple times.
