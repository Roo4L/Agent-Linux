<objective>
Add rtk as the first prebuilt-binary catalog entry (WORK-02) and write its install/uninstall recipe pair against the Plan 02 shared helper. Install fetches `rtk-ai/rtk@0.42.4` (never the crates.io "Rust Type Kit" collision — `cargo install rtk` is never used), checksum-verifies, lands `rtk` in `~/.local/bin`, asserts the pin, and prints the OPT-IN instruction (`rtk init -g`) WITHOUT mutating `~/.claude`. Uninstall reverts rtk's own Claude Code hook first (`rtk init --uninstall`, while the binary still exists), then deletes the binary + config/cache, idempotently.

Purpose: rtk is the first consumer that proves ENABLE-01 end-to-end and the opt-in/symmetric-remove contract (WORK-02 success criteria 2-3-4). The opt-in ordering is load-bearing: deleting the binary before reverting the hook would orphan a `rtk hook claude` entry in `~/.claude/settings.json` and break every Bash tool call in Claude Code (RESEARCH Pitfall 4).
Output: a validated rtk entry in catalog.json + a recipe pair that drives install/remove via the shared helper, ready for the Plan 04 bats lifecycle test.
</objective>

         `if command -v rtk >/dev/null 2>&1; then rtk init --uninstall -g

         user never opted in.)
      2. `rm -f "${AGENTLINUX_AGENT_HOME}/.local/bin/rtk"`.
      3. `rm -rf "${AGENTLINUX_AGENT_HOME}/.config/rtk"

         remove these).

         backup residue). Do NOT delete ~/.claude/settings.json itself (it is
         user-owned; rtk init --uninstall leaves an empty PreToolUse scaffold).
      5. `hash -r`; assert rtk is gone:
         `command -v rtk >/dev/null 2>&1 && { echo "rtk uninstall: still on
         PATH" >&2; exit 1; } || true`; `echo "rtk: uninstall complete"`.
    Every destructive step is guarded / `|| true` so a second remove on a missing
    install exits 0 (idempotent).
  </action>
