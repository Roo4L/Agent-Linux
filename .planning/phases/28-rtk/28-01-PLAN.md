<objective>
Extend the catalog's `source_kind` enum with a third value, `"binary"`, in both the JSON schema (the ajv source of truth) and the TypeScript `CatalogEntry` union, and prove the change with unit tests. This is the ONLY CLI/TypeScript-surface change ENABLE-01 requires — `runner.ts`, `upgrade.ts`, `loader.ts`, and the provisioner all already handle any non-npm source_kind generically.

Purpose: Without `"binary"` in the enum, ajv rejects the rtk catalog entry at load time and the whole prebuilt-binary machinery is unreachable. Per RESEARCH.md (§Architectural Responsibility Map, §Schema delta) this is a one-line enum extension mirrored in two files plus test coverage.
Output: schema.json + types.ts accept `source_kind: "binary"`; schema.test.ts proves a binary entry validates and that the enum error lists `binary`.
</objective>
