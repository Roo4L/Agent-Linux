<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller → pkg verbs | Package names ("$@") flow into apt-get/dnf command words |
| network → nodesource_setup | curl-pipe-bash of a remote setup script executes as root |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-05 | Injection | pkg_install/pkg_remove "$@" → apt-get/dnf | mitigate | All call-site args are static in-code literals (jq, nodejs, sudo, ca-certificates); "$@" is quoted; no untrusted/user-derived string reaches a verb in this phase |
| T-18-06 | Tampering | nodesource_setup curl-pipe-bash (rhel: rpm.nodesource.com) | mitigate | HTTPS + `curl -fsSL` cert verification is the fetch integrity control; ongoing package integrity comes from the repo's `gpgkey=`+`gpgcheck=1` (set by setup_22.x). Same posture as ADR-005 (deb path); script-body SHA not published — accepted per ADR-005 |
| T-18-07 | Tampering | locale_ensure write to /etc/locale.conf | mitigate | Use write_file_atomic (mode 0644, install(1) atomic rename) — no partial/world-writable window; content is two fixed LANG/LC_ALL lines, no interpolation of untrusted data |
| T-18-08 | DoS | dnf install_weak_deps=False dropping a runtime dep | accept | Mirrors `--no-install-recommends`; the RT-01 node>=22 hard-check (plan 03) catches an incomplete Node install on the Docker arm |
</threat_model>
