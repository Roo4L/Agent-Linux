<threat_model>
- Privilege escalation: only OS package installation may use `sudo -n`; npm,
  uv, browser caches, and config files remain agent-owned. Password failure is a
  clear nonzero error, never a silent help/version fallback.
- Package integrity: use distro package-manager transaction/checks and official
  Playwright/Chrome sources; do not execute arbitrary downloaded scripts as root.
- Ownership and residue: browser caches and skill/config trees are created as
  agent, and uninstall removes only AgentLinux-managed artifacts while preserving
  user browser/profile data according to preserve_paths.
- Status propagation: do not swallow Playwright action errors; test representative
  target-resolution families and a successful action.
- Prerequisite detection: validate executable paths and shared libraries with a
  real browser operation, not only `--version`.
</threat_model>

    `/opt/google/chrome/chrome`; if the repository is unavailable, fail with the
    escalation message rather than silently relying on a different browser.
    Make repeated calls idempotent and keep all downloaded/cache files owned by
    the agent. Add shellcheck-safe quoting and no broad recursive deletion.
  </action>
