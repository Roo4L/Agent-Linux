# Phase 37 Plan — sentry-mcp (hosted remote, thin installer)

**Requirement:** MCP-04 · **Convention:** ADR-017 thin installer (bare URL, no
credential, in-client auth). **Reuses:** the Phase 36 remote-http cross-agent
helper (now credential-free).

## Goal (goal-backward)

`agentlinux install sentry-mcp` registers the bare `https://mcp.sentry.dev/mcp`
into every installed MCP-capable agent (no credential stored); `remove`
deregisters symmetrically with no residue; the FSL-1.1-ALv2 license is recorded;
≥1 bats @test proves the lifecycle + no-credential + no-Docker — Docker green.

## Task breakdown

### 37-01 — recipe pair + catalog entry
- `plugin/catalog/agents/sentry-mcp/install.sh` — source the shared helper,
  `al_mcp_register_http sentry-mcp https://mcp.sentry.dev/mcp`, friendly no-agent
  error, in-client-auth (Sentry OAuth) note. `uninstall.sh` — deregister + assert
  absent.
- `catalog.json` — `source_kind:mcp`, `pinned_version:0.37.0`, `endpoint_url`,
  `requires_secret:true` (NO `secret_env`), `license:FSL-1.1-ALv2`,
  `post_install_verify` across the 5 agent configs.

### 37-02 — helper retrofit to ADR-017 (shared, also covers github-mcp)
- `plugin/catalog/lib/mcp-register.sh`: `al_mcp_register_http` drops the
  `<secret_env>` param and all header/reference/never-bake machinery — registers
  the bare URL into each agent (claude `--transport http` no header; codex block
  `url` only; gemini/qwen bare `httpUrl`; opencode `type:remote url`). Landed-entry
  checks replace the reference assertions.
- Retrofit `plugin/catalog/agents/github-mcp/{install.sh,catalog entry}`: bare-URL
  register, drop `secret_env`, in-client-auth note. Rewrite
  `tests/bats/60-catalog-github-mcp.bats` to assert bare-URL + NO credential.

### 37-03 — bats gate + docs + planning
- `tests/bats/61-catalog-sentry-mcp.bats` (MCP-04): bare-URL fan-out into
  claude+codex + conditional gemini/opencode/qwen, no-credential grep across all 5
  configs, symmetric residue-free removal, idempotent re-remove, entry-shape +
  no-Docker check.
- `docs/decisions/017-*.md` (the ADR), `docs/internals/catalog.md` (thin-installer
  rewrite of the hosted-server section + roster), REQUIREMENTS/ROADMAP/STATE.

## Verification (TST-07 / OPS-01)
- Docker Ubuntu 24.04 green; MCP-04 @tests + the retrofitted MCP-03 @tests pass.
- OPS-01 note: a live Sentry OAuth smoke needs a human in a browser — out of scope
  for headless CI; the registration-shape lifecycle is credential-free and fully
  covered.

## Threats
- T-37-01 a credential accidentally written to a config → no-credential grep
  (Authorization/bearer/token/secret/api-key) across all 5 configs.
- T-37-02 non-atomic config write corrupts an agent's settings → mktemp+mv, parse
  guard leaves original on failure (inherited from the helper).
