<objective>
Land the Phase 14 foundation: `--yes` / `--no-yes` argv parsing, structured exit code constants (`EX_USAGE=64`, `EX_DATAERR=65`), the `--help` "Exit codes" section, AND — critically — the **DECIDE-THEN-ACT** orchestration that makes the non-TTY-without-`--yes` contract actually enforceable.

**The architectural shape this plan establishes:**

```
parse_args                                # --yes / --no-yes; EX_USAGE on bad flags
detect::run_once                          # populate DETECT_* exports (Phase 12 — read-only)
remediate::collect_all_decisions          # call EVERY reuse::*_decision; populate RESOLUTIONS map + BAILED_COMPONENTS array; NO MUTATION
remediate::flush_bails_or_continue        # if BAILED_COMPONENTS non-empty AND non-TTY AND !--yes → print [BAIL] lines + exit 65; OTHERWISE return 0
run_provisioners                          # each provisioner reads RESOLUTIONS[<component>] and dispatches; bail arm is unreachable (gate already exited)
```

This DECIDE-THEN-ACT flow is the only design that satisfies UX-03's "non-TTY without --yes never overwrites user state" contract. The previous design — embedding decision calls inside provisioner case-branches — could not guarantee atomicity: by the time provisioner N's bail registered, provisioners 1..N-1 might have already mutated the host. With DECIDE-THEN-ACT, the gate runs in deterministic main() position BEFORE any provisioner; on bail, zero mutations have happened.

Phase 13's `remediate)` and `bail)` dispatch arms — currently `return 1` placeholders — get replaced with handler stubs that downstream Plans 14-02 + 14-03 fill. Reuse decision functions are tightened to distinguish fixable defects (return `remediate`) from unfixable defects (return `bail`) so the dispatcher routes into the right policy gate.

This plan owns UX-03 (the `--yes` non-interactive default contract) and UX-05 (the structured exit codes contract). Plan 14-02 fills the REMEDIATE-01/02/03 handler bodies; Plan 14-03 fills the REMEDIATE-04 CLI + sentinel path.

Purpose: The dispatch surface Phase 13 locked (`case "$(reuse::*_decision)" in reuse|create|remediate|bail`) needs the bail-aggregation policy gate AND the DECIDE-THEN-ACT main() restructure before real Remediate handlers can land — without both, a host with two fixable defects would partially install + half-fail mid-execution, leaving the operator with an ambiguous host state. The `--yes` flag must be the SOLE consent surface (no per-action flags, no env-var spoofs per T-14-01) and the exit codes must be stable from day one so CI wrappers can branch on them across Plans 14-02 / 14-03 / Phases 15-16.

Output: A working `agentlinux-install --yes` flag, a working bail message format with proven zero-mutation atomicity, EX_USAGE/EX_DATAERR exits at the right surfaces, the DECIDE-THEN-ACT main() flow, and a `tests/bats/14-remediate.bats` covering the foundation contract end-to-end (including byte-equal snapshot tests proving non-mutation on bail). Plan 14-02 then drops in handler bodies behind this gate.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator CLI argv → entrypoint | Operator passes flags; entrypoint MUST reject unknown/contradictory flags with EX_USAGE rather than silently ignoring (security-by-default: a typo of `--yes` should not silently disable consent) |
| operator environment → entrypoint | Operator-controlled env vars MUST NOT spoof consent (T-14-01) |
| detect::* readers → reuse::*_decision → remediate::register_bail | Detected host state drives the bail aggregation; tampered detect output could trigger an unintended `[BAIL]` line, but the bail itself is non-destructive (exits 65, writes nothing) |
| collect_all_decisions → flush_bails_or_continue → run_provisioners | Hard ordering invariant — no provisioner runs until after the gate. Verified by bats no-mutation snapshot tests. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-01 | E (Elevation of Privilege) | `plugin/bin/agentlinux-install` argv parser | mitigate | `--yes` is the SOLE consent surface. The entrypoint reads ZERO env vars named `AGENTLINUX_YES`, `ALWAYS_YES`, `ASSUME_YES`, `CONFIRM`, or any synonym — verified by a bats @test that greps the entrypoint source returning 0 matches for `AGENTLINUX_YES`/`ALWAYS_YES`/`ASSUME_YES`. The `--help` Exit codes section explicitly documents "consent is via `--yes` only; no env-var equivalent exists." |
| T-14-02 | T (Tampering) | `plugin/bin/agentlinux-install` argv parser | mitigate | Contradictory flag combos (`--yes --no-yes`, `--no-yes --yes`) MUST exit 64. The last-flag-wins fallback is rejected explicitly — argv must be unambiguous. Verified by a bats @test asserting exit 64 + a log_error line. |
| T-14-03 | I (Information Disclosure) | bail message format | accept | The `[BAIL] component=X reason=Y hint=Z` lines disclose which components are in a non-compatible state. Accepted per CONTEXT.md Area 1 Q2. |
| T-14-06 | S (Spoofing) | provisioner-side `remediate::register_bail` call sites | mitigate | Each register_bail call site uses a hardcoded literal component name (`npm-prefix`, `sudoers`, etc.) — never a `$VAR`-driven value sourced from detect:: output. Verified by a bats @test grepping plugin/lib/remediate/*.sh + plugin/provisioner/*.sh for `register_bail "\$` (zero matches expected). |
| T-14-13 (new) | T (Tampering) | DECIDE-THEN-ACT ordering invariant | mitigate | The bats no-mutation snapshot tests (Tests 18-21 below) take byte-level snapshots of /etc/sudoers.d/, /home/, /etc/passwd BEFORE running `agentlinux install --no-yes` on a bail fixture, then again AFTER, and assert byte-equality. A regression that re-orders main() to put run_provisioners BEFORE flush_bails_or_continue would be caught immediately. This is the real proof of UX-03's "non-TTY without --yes never overwrites user state" contract. |

         ```bash
         # exactly once, populate RESOLUTIONS[<component>]=<token>, and (where the
         # token maps to a state-overwriting remediate AND no --yes) register a bail.
         #

         # exports and calls reuse::*_decision functions which themselves are pure
         # readers. Verified by Test 11 (filesystem-mutation shim).
         #
         # After this returns, downstream consumers (run_provisioners) read
         # RESOLUTIONS[<component>] and dispatch — they do NOT call reuse::*_decision
         # themselves (decisions are pre-resolved here).
         remediate::collect_all_decisions() {
           local user=${INSTALL_USER:-agent}

           # 1. User decision (REUSE-01 / unfixable bails)
           RESOLUTIONS[user]=$(reuse::user_decision "$user")
           case "${RESOLUTIONS[user]}" in
             bail)
               remediate::register_bail \
                 "user" "incompatible" \
                 "use --user=NAME with a compatible user, or fix shell/home of existing user"
               ;;
             remediate)
               # The user-decision "remediate" maps to REMEDIATE-03 (sudoers fix).
               # That fix is OWNED by the sudoers component below, NOT by the user
               # component. We record the resolution but do NOT register a bail
               # here — the sudoers branch will handle gating.
               :
               ;;
           esac

           # 2. Node decision (Phase 13 — reuse|create only; no remediate token here)
           RESOLUTIONS[node]=$(reuse::nodejs_decision)

           # 3. npm-prefix decision (Plan 14-01 NEW — REMEDIATE-01)
           RESOLUTIONS[npm-prefix]=$(reuse::npm_prefix_decision)
           if [[ "${RESOLUTIONS[npm-prefix]}" == "remediate" ]]; then
             remediate::gate_or_bail \
               "npm-prefix" "npm-prefix-chown" "wrong-owner" \
               "run with --yes to chown or rebase" \
               || true   # return-1-on-bail is OK here; bail is now in BAILED_COMPONENTS
           fi

           # 4. Sudoers decision (inline — small enough; see CONTEXT.md Area 1 Q1)
           local sudoers_token=create
           if [[ "${DETECT_SUDOERS_PRESENT:-false}" == "true" ]]; then
             if [[ "${DETECT_SUDOERS_NOPASSWD_OK:-false}" == "true" ]]; then
               sudoers_token=reuse
             else
               sudoers_token=remediate
             fi
           fi
           RESOLUTIONS[sudoers]=$sudoers_token
           if [[ "$sudoers_token" == "remediate" ]]; then
             remediate::gate_or_bail \
               "sudoers" "sudoers-drift-overwrite" "drift" \
               "run with --yes to overwrite with the canonical ADR-012 line" \
               || true
           fi

           # 5. Per-agent decisions (Phase 13 — iterate REUSE_AGENT_CANONICAL_PATHS keys)
           local agent_id
           for agent_id in "${!REUSE_AGENT_CANONICAL_PATHS[@]}"; do
             RESOLUTIONS[agents.$agent_id]=$(reuse::agent_decision "$agent_id")
             if [[ "${RESOLUTIONS[agents.$agent_id]}" == "remediate" ]]; then
               remediate::gate_or_bail \
                 "$agent_id" "agent-reinstall" "broken-or-path-mismatch" \
                 "run with --yes to uninstall and reinstall (preserves user data per CAT-04)" \
                 || true
             fi
           done

           return 0
         }
         ```
