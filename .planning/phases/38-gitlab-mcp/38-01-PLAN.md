# Phase 38 Plan — gitlab-mcp (official hosted, thin installer)

**Requirement:** MCP-05 · **Convention:** ADR-017 thin installer (bare URL, no
credential, in-client auth) + source-selection policy (prefer first-party). Reuses
the credential-free ENABLE-02 helper.

## Goal (goal-backward)

`agentlinux install gitlab-mcp` registers the bare `https://gitlab.com/api/v4/mcp`
into every installed MCP-capable agent (no credential); `remove` deregisters
symmetrically; ≥1 bats @test proves the lifecycle + no-credential + first-party
(no docker/npx/third-party) shape — Docker green.

## Tasks

### 38-01 — recipe pair + catalog entry + bats + docs
- `plugin/catalog/agents/gitlab-mcp/{install,uninstall}.sh` — mirror sentry-mcp;
  url `https://gitlab.com/api/v4/mcp`; in-client-auth (GitLab OAuth) note +
  self-managed pointer.
- `catalog.json` — source_kind mcp, pin `18.6.0` (GitLab release), endpoint_url,
  `requires_secret:true` (no `secret_env`), `license:MIT`, 5-agent
  post_install_verify.
- `tests/bats/62-catalog-gitlab-mcp.bats` (MCP-05) — bare-URL fan-out into
  claude+codex + conditional gemini/opencode/qwen (+ __diag skips), tight
  no-credential grep (incl. `glpat-`/`PRIVATE-TOKEN`), symmetric residue-free
  removal, entry-shape + first-party-only (no docker/npx/zereight) check.
- ADR-017 addendum (source-selection policy) + REQUIREMENTS/ROADMAP/STATE +
  catalog.md roster.
